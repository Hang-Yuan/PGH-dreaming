# dream

夜间 cron 无人值守触发的记忆代谢。按 1→8 顺序执行。判据本体全在 `<ASSISTANT_ROOT>\MEMORY\00.memory_agent.md`，本 skill 只走指针 + 落无人值守执行编排。

## 无人值守总纪律（贯穿全程）

- **不阻塞**：任何步骤不停下等回复——无人在场，停 = 整夜作业卡死。
- **需 C verdict 的只生成候选、绝不执行**：毕业写入 / 季度归档等 C 级动作只记候选挂出口，等 [用户称呼] 裁。
- **只走免审批操作**：Bash 追加 / 既有文件 Edit / 派 storage-agent。会弹权限的操作一律不进夜班（弹窗在无人值守下永远得不到响应）。
- **失败留可观测痕迹**：中途任何步骤失败 → 不进第 8 步关机；"机器还开着 + MEMORY_LOG 缺当晚 dream 条目"被设计成次日可一眼看出的故障信号。

## 输入
- L0 = 当日会话转写 jsonl（`<CLAUDE_HOME>\projects\` 各端目录），唯一回放源。
- 判据权威源 `00.memory_agent.md`；排梦任务包（载荷标记 / 逻辑日期 / 关机指令）。

## 产物
- 更新后的 `episodic_memory.md` / `semantic_memory.md` 两池（semantic = 中间工作区，不注入，仅夜间被本 skill 读写）。
- MEMORY_LOG 一条 dream 汇总账（派 storage-agent）+ `last_dream.md` 探针覆盖 +（待 C 项）挂 `<ASSISTANT_ROOT>\长期记忆.md §当前处境` 的「上次未决」。

---

## 第 1 步 · 定位输入

按 `00.memory_agent.md §逻辑日期` 口径定当日，glob `<CLAUDE_HOME>\projects\` 下当日修改的 jsonl（覆盖全部会话端）。

- **回放深度自判**：重日全量、轻日只扫尾段——以当日 `_本周` 流水段数为判据。
- **补扫模式**：按排梦包指定日期段回放；上限见 `§L0 回放提取规则`（更早接受信号损失 = 默认遗忘）。

---

## 第 2 步 · 容量扫描（开工必做 · 不可跳过）

按 `00.memory_agent.md §两池容量阈值` 扫两池条数 → 报数 + 黄区提示 / 红区触发当次清理。结果写进本晚 MEMORY_LOG dream 条目。

- **扫描对象**：episodic 活动+复审 / 候补 / 休眠；semantic **活跃组**（v5.6.0 退注入后口径为「活跃组」，非「启动注入组」）+ 组内项目语境条目硬限。
- **职责衔接**：本步只报数 + 红区"触发判定"；红区的实际清理（episodic 衰减清到 ≤40）由第 6 步周日载荷执行（D 类衰减唯一执行者是周日梦）。平日红区只在 MEMORY_LOG 报"超红区待周日清"，不在平日强清。

> 前置在代谢之前：先知道池子多满，才知道这一晚要不要顺手清。

---

## 第 3 步 · 分段回放

派 storage-agent 按块读当日转写 → 返回候选信号清单（带两轴预标 + 原文锚点）。

- **IO 与裁决分离**：重 IO 派 storage-agent、可指定低成本模型；**两轴裁决留主循环**（判断密度不下放）。
- **对照基线必须在场**：两轴判定是差分运算、不是孤立打分（`§代谢对照基线`）——回放裁决前，现有 schema 全集（episodic 主文件 + semantic 主文件 + 已注入的身份层）须作"差分的减数"在场，否则判不出"命中已有"还是"全新"。semantic 退注入后，**代谢期是 semantic 唯一被读取的时机**，本步读 semantic 主文件作基线是其设计闭环的必要动作。
- **校验事件走向**：回放时核对当日事件是否已被在场段（close-node / daily-review 阶段 A）固化进工作库——**漏账只报不补写**（记入 MEMORY_LOG），不越界替白天补工作库。

---

## 第 4 步 · 判断节点 · 两轴判定

对每条候选信号按 `00.memory_agent.md §两轴判定` 裁决：P轴（预测关系：命中/击穿/未覆盖）× C轴（校准：正向/反对/纠正/中性）。

- **路由先过 `§身份层前置过滤`（关键：先看命中的是哪一层）**：
  - 命中**终态层**（USER / SOUL）+ 中性**或正向确认** → **不提取**（已毕业终态，符合旧认知不重复加权）。
  - 命中终态层 + 抵触 / 新扰动 / 击穿 → 进代谢判定，**不直接改身份层**（身份层只接受 close-node / dream 受控写入）。
  - 命中**中间态**（semantic 活跃组）+ 正向确认 / 命中 → **提取**（archive 补证据、按 `§升星` 加权——未毕业 schema 的爬星燃料）；击穿 → 标记修订（→ `§semantic 衰减`）。
  - 未命中身份层 / semantic（全新信号）→ 走 `§提取过滤下限`：`命中+中性 → 不提取；其余组合 → 进代谢`。
- **判准核心**：同样是「命中+正向确认」，命中终态不提取、命中中间态要提取——差别在这一层是不是还在路上。通过的进第 5 步代谢。

---

## 第 5 步 · 判断节点 · 日级代谢

对过两轴的信号按 `00.memory_agent.md §生成` + `§升星` + `§L0→L1 升格抽象红线` 代谢：

- **判断标准**：信号对现有 episodic 是升星（同结构）/ 新建 / 候补，看 ≥2 独立事件红线 + 深层结构查重（对照基线 = episodic 主文件）。
- **路由**（按判准分流，逐条判）：
  - 升星节律一律按 `§升星`（同日只计一次、7 工作日窗口、跨情景 ★★★ 再命中的去向等全以该节为准，本处不内联）。
  - ≥2 独立事件支撑 → 升活动态；单事件值得追踪 → 候补态 ★；单事件换句式 → 不升。
  - 项目语境信号 ≥3 工作日复现 + 过判准 → 走 `§项目语境快轨` 升 semantic 活跃组（强制标注语境）。日级直升仅此 + 高风险/越权/系统事故单条（`§升星` 两例外），不经横向统合门。
  - 跨情景信号达 ★★★ 再命中 → **不日级直升**，转标 `待统合簇` 留周日横向统合判去向（`§横向统合` 是跨情景升 semantic 唯一门）。
- **semantic 写动作授权**：dream 对 semantic 的升格 / 合并 / 标修订 / 衰减全部是 **N 级**（`§写入授权` semantic 全部 dream 自治），做完 MEMORY_LOG 留账即可，不需 C verdict。
- **执行顺序硬约束**：升格 / 统合一律 `§升格动作` 第 3 条——本体 + 证据**先**落盘，确认后**才**物理删贡献来源（先固化后删除，防夜间中途崩导致信息净丢失）。

---

## 第 6 步 · 判断节点 · 周日载荷（条件触发）

- **判断标准**：本晚排梦包是否带「周日梦载荷」标记（daily-review 第 3 步按逻辑日周日排定）。
- **触发钩子**：带标记 → 跑；不带（平日）→ 整步跳过。
- **路由（带标记才执行，全按 memory_agent 对应节）**：
  1. episodic 衰减（`§episodic 衰减`，周日梦是 D 类衰减唯一执行者；红区必须清到 ≤40）。
  2. 横向统合（`§横向统合`，按深层结构聚类析母结构，防 over-merge）。
  3. semantic 审查 + 衰减（`§semantic 衰减`，含活跃组降级回 episodic 的去向分流：跨情景条目 vs 项目语境条目去向不同）。
  4. 升格候选池重扫（清空：升 semantic 活跃组 / 退回 episodic / 删，均 N 级）。
  5. **毕业候选生成**（`§判型分流` + `§三路独立蒸馏`）：判型定出口 → 拉 3 个互不可见 sub-agent 独立蒸馏 → 收敛判定（三路一致才收敛）。hook 候选追加 `§实测验证`（**安全红线**：收敛版存暂存区 `<ASSISTANT_ROOT>\MEMORY\_dream_staging\`，全程不触 `settings.json`、不进 hooks 生效目录；3 sub-agent 沙盒实跑全过才升"已验证"）。程序型 skill 出口 = 挂"[用户称呼] 授权→启用 meta-skill 蒸馏新 skill"的候选，不在夜间直接写 skill。
  6. **毕业只生成候选、不写身份层**——USER/SOUL/skill/§R/hook 写入是 C 级，候选挂出口等 C verdict（无人值守不执行不可逆身份层写入）。
  7. 周归档相关（账实核对等）见 weekly-review 周日梦载荷定义。

> 第 6 步是 dream 判断节点最密的一步；平日空过，周日全跑。

---

## 第 7 步 · 巡检（代谢主干末端收尾扫描）

代谢跑完顺手扫系统卫生（dream 是唯一夜间无人值守班车，巡检无独立载体故附此趟）。逐项扫、异常记入 MEMORY_LOG dream 条目：

- **锚点死链**：随机抽 5-8 个节锚点 grep 验证可达，死链报出。
- **frontmatter 一致性**：`updated` 与当日实际改动抽查（防"改了没更新 / 没改却动了时间戳"漂移）。
- **progress 转换标记完整性**：活跃项目 `_progress` 扫断头节点（开了问题没收口 / 转向模式未消费）。
- **季度审查月**：额外做 episodic 休眠态整批出清（`§episodic 状态机`）。

巡检是收尾扫描、不是代谢——只报 + 轻量清，不在此做重判断。

---

## 第 8 步 · 落账 + 关机

1. **MEMORY_LOG dream 条目**（派 storage-agent，`§MEMORY_LOG 收录边界`）：一条汇总流水（升格 / 删 / 合并 / 升降星 / 衰减 / 唤醒 / 统合 + 容量报数 + 巡检异常），明细在条目内，不另起逐条流水、不另写晨报文件。两 log 读写一律 storage-agent。
2. **待 C 裁决项**（毕业候选 / 双权威源冲突）→ 挂 `<ASSISTANT_ROOT>\长期记忆.md §当前处境` 的「上次未决」一行（启动必读，保证每会话可见直到处理）。
3. **覆盖 `MEMORY/last_dream.md`**：一行逻辑日期（纯机器探针，仅此一行），供次日 week-sync 昨梦核对。
4. **关机（唯一且最后一步）**：默认**不关**（跑完留机）；仅排梦包显式带关机指令才执行。中途任何步骤失败 → 不到这步 = 不关机（故障信号见总纪律）。

---

## 边界
- 不在白天被调用（白天唯一记忆动作 = close-node 节点级直升）。
- 不写工作库（事件走向归在场段，dream 只校验不补写）。
- 不直接 Read/Edit 两 log（一律派 storage-agent）。
- 不动 hook / settings / 协议层；毕业候选只挂出口，C verdict 后才由在场段（USER/SOUL）或 meta-skill 通道（skill）写身份层。
