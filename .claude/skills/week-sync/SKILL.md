# week-sync

专注区状态同步权威源。由 `CLAUDE.md §B 启动序列` 步骤 6 每次会话启动时自动调用。每天统一轻量档——启动侧只做便宜的读取、简报与提醒；贵的审计与代谢全部归夜间 dream。

## 加载链

**上游**：`CLAUDE.md §B 启动序列` 步骤 6 — 每次会话自动调用。

**同级联动**：`weekly-review`（周日提醒的执行去向）/ `dream`（昨梦核对的补扫去向）。

**下游**：`<ASSISTANT_ROOT>\00 专注区\_本周.md`（只读，不修改）/ `MEMORY/last_dream.md`（只读，一行日期探针）。

---

## 同步流程（每天统一）

1. 读取 `_本周.md`，确认当前周次、任务清单和进展记录。
2. 向用户简报：本周第几天、任务完成状态（X/N 完成）、近期进展一句话摘要。
3. 执行下方三项检查。
4. 不修改文件，不询问细节。

---

## 检查一 · 周日复盘提醒（仅周日）

1. 检查 `_本周.md` 是否已有 `### 本周产出` 节。
2. **有** → 本周复盘已完成，不提。
3. **无** → 在同步报告末尾提示：「今天是周日，本周复盘还没做。要现在开始吗？」（等用户回应，不自动触发；用户确认后执行 `weekly-review` skill）

## 检查二 · 昨梦核对（每次必须执行）

读 `<ASSISTANT_ROOT>\MEMORY\last_dream.md`（一行 = 最近一次梦的逻辑日期）：

1. **日期核对**：探针日期 vs 最近有效工作日。断档 → 提示[用户称呼]补扫派单（dream 补扫上限 3 个有效工作日）。
2. **当日首会话**（今日 `_本周.md` 尚无流水段）→ 派storage-agent读 `MEMORY_LOG.md` 最新 dream 条目，回报两三行摘要（升格·升星·删除 + 巡检异常 + 待 C 裁决项）；非首会话 → 只报「昨梦正常（日期）」。
3. 待 C 裁决项的常驻出口 = `长期记忆.md §当前处境` 上次未决（启动必读自然带出），本检查不重复维护。

## 检查三 · 接续检查（每次必须执行）

1. 读取 `_本周.md` 进展记录，找到**最近一次有实质工作的日期**（跳过空白日）。
2. 从该日期节中提取所有工作段的：项目名称 + 「关联：」字段中的文件列表。
3. 在报告末尾附上接续提示：

```
---
**上次工作**（YYYY-MM-DD）：
- [项目名]：[当天做了什么，一句话]
  → 关联文件：[文件名1] · [文件名2]

要继续的话，告诉我，我先读一下相关文件再给你汇总。
```

> 如果用户当天第一句话就是「继续」「继续昨天」「从哪里开始」等，直接读取关联文件，给出完整汇总，不等确认。

---

## 不做

- 不做差集扫描 / 漏记文件审计 / 任务状态核对——归 `weekly-review` §周日梦载荷（账实核对项）。
- 不修改任何文件；不自动触发 weekly-review / dream 补扫（只提示，等确认）。
