# week-sync

会话启动同步。按 1→7 顺序执行，不得跳步。只读不写——本流程不修改任何文件。

## 输入

- `<ASSISTANT_ROOT>\00 专注区\_本周.md`（周次 / 任务清单 / 进展记录）。
- `<ASSISTANT_ROOT>\MEMORY\last_dream.md`（一行 = 最近一次梦的逻辑日期）。
- 启动序列已读入的当前处境（待 C 裁决项常驻出口在那里，本流程不重复维护）。

## 产物

一份启动简报，含三段：本周进度 + 昨梦核对 + 接续提示；周日且未复盘时追加一行复盘提醒。

---

## 第 1 步 · 读本周状态

读 `_本周.md`，取周次、任务清单、进展记录。

---

## 第 2 步 · 报本周进度

输出一句：本周第几天 + 任务完成 X/N + 最近进展一句话摘要。

---

## 第 3 步 · 判断节点 · 周日复盘提醒

**仅周日跑，非周日跳过本步。**

- **判断标准**：周日当天，本周复盘是否已做——以 `_本周.md` 有无 `### 本周产出` 节为唯一判据。
- **触发钩子**：今天是周日 且 `_本周.md` 无 `### 本周产出` 节。
- **路由**：
  - 触发 → 在报告末尾提示「今天周日，本周复盘还没做，要现在开始吗？」**等用户回应，不自动触发** `weekly-review`。
  - 已有 `### 本周产出` 节 → 复盘已完成，不提。
  - 非周日 → 跳过本步。

---

## 第 4 步 · 读梦探针

读 `last_dream.md`，取最近一次梦的逻辑日期。

---

## 第 5 步 · 判断节点 · 梦断档核对

- **判断标准**：探针日期对齐最近一个有效工作日则正常；落后则断档。"有效工作日"定义见 `<ASSISTANT_ROOT>\MEMORY\00.memory_agent.md §有效工作日`，从 `_本周.md §进展记录` 有流水追加的日子回推。**基准 = 今天之前（不含当天）最后一个有效工作日**——今日首会话尚无流水属正常，不计入，避免把首会话早晨误判为断档。
- **触发钩子**：探针日期 < 不含当天的最近有效工作日。
- **路由**：
  - 断档 → 提示 [用户称呼] 派 dream 补扫；补扫回放上限见 `00.memory_agent.md §L0 回放提取规则`（不在本流程内重述数值，指针为准）。**只提示，不自动派单。**
  - 对齐 → 继续。

---

## 第 6 步 · 判断节点 · 昨梦摘要深度

- **判断标准**：今日是否首次会话——以 `_本周.md §进展记录` 今日有无流水段为判据。
- **触发钩子**：今日 `_本周.md` 尚无当日流水段 = 首会话。
- **路由**：
  - 首会话 → 派 storage-agent 读 `MEMORY_LOG.md` 最新 dream 条目，回报两三行摘要（升格 / 升星 / 删除 + 巡检异常 + 待 C 裁决项）。两 log 读取一律派 storage-agent，见 `00.memory_agent.md §写入授权`。
  - 非首会话 → 只报「昨梦正常（日期）」。
- **断档态修正**（接第 5 步）：第 5 步判为断档时，读到的"最新 dream 条目"是断档前最后一梦、非真正昨梦 → 摘要须标注「断档前最后一梦（日期）」；非首会话分支此时不得报「昨梦正常」，改报「梦断档，最近一梦为（日期）」。

> 待 C 裁决项的常驻出口 = `<ASSISTANT_ROOT>\长期记忆.md §当前处境` 上次未决（启动必读自然带出），本步不重复维护。

---

## 第 7 步 · 判断节点 · 接续

- **判断标准**：用户当天首句是否表达「从上次工作继续」的意图。
- **触发钩子**：首句匹配「继续 / 继续昨天 / 从哪里开始」类接续语。
- **关联文件**（两分支共用定义）：取第 1 步进展记录里最近一次有实质工作的日期（跳过空白日），提取该日各工作段的项目名 + 关联文件列表。
- **路由**：
  - 触发 → 跳过提示，直接读上述关联文件给完整汇总。
  - 未触发 → 在报告末尾附接续提示（上述项目名 + 关联文件列表），等用户确认再读文件。

接续提示格式：
```
---
**上次工作**（YYYY-MM-DD）：
- [项目名]：[当天做了什么，一句话]
  → 关联文件：[文件名1] · [文件名2]

要继续的话告诉我，我先读相关文件再汇总。
```

---

## 边界

- 不做差集扫描 / 漏记文件审计 / 任务状态核对——归 `weekly-review` 周日梦载荷（账实核对项）。
- 不修改任何文件；不自动触发 `weekly-review` / dream 补扫（只提示，等确认）。
