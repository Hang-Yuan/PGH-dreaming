# close-node

## 加载链（上下游）

- **上游**：AI 自判触发 / daily-review 节点闭合兜底 / [用户称呼]明确要求节点收束。
- **规则内核**：`<ASSISTANT_ROOT>\MEMORY\00.memory_agent.md` — 代谢判准 / 红线 / 授权的单一权威源，执行阶段 B 前按需读取。
- **管辖文件（下游）**：当前项目主文档 / `_overview.md` / `_progress/` / `episodic_memory.md`（仅 ≥2 事件直升）。
- **同级联动**：`write-progress`（步骤 4 调用；转换标记定义权威源）/ `daily-review`（日级兜底）/ `dream`（单次信号的夜间提取者）/ `new-file`（新建文件补加载链）/ `storage-agent.md`（log 落盘派单 + 格式护栏）。

---

## 定位

节点闭合时的多文件同步流程，比 daily-review 粒度细。**粒度参数：节点级**——这是工作结论的小时级固化时刻（→ `00.memory_agent.md §代谢速度梯度`）；记忆动作仅限 ≥2 独立事件直升 episodic，单次信号零动作（当晚 dream 从转写提取）。

边界：

- In_Progress / `_progress` 是项目推论链权威源，不是记忆写入落点。
- 段间转换节点标 `→ 可提取转向模式`（标记定义见 write-progress §2.3 追加与扩展），周日梦提取，close-node 不直接升。
- 不依赖 hook，不改 hook。

---

## 触发

AI 自判。识别信号：连续讨论 / 工作收束、子问题解决、决策拍板、任务完成、用户说「好了 / 收 / 下一个 / 这块告一段落」。

触发时 N 级提议：「这段按节点闭合处理？涉及 [项目X]。」确认后执行；用户已明确授权当前项目闭合时直接执行；daily-review 告别场景兜底补跑时不提议、直接执行（无人值守，按步骤 2 无人值守例外处理 verdict 类动作）。

---

## 流程总览（两阶段）

- **阶段 A · 当前工作的归档**：步骤 1-4（核心项目同步）+ 步骤 6-8（收尾）。
- **阶段 B · 记忆的归档**：步骤 5（节点级分流）。物理位置紧接步骤 4，是为趁项目结论刚定稿做分流；逻辑上仍是"工作产出先固化、再代谢记忆"。

## 阶段 A · 当前工作的归档

### 步骤 1 · 项目定位

清楚 → 继续；模糊 → 向[用户称呼]确认；跨项目 → 选主要权威落点，其他项目只留指针，跨项目影响按 C 级确认。

### 步骤 2 · 主文档（强制判断 + C 级 verdict）

**硬约束**：每次 close-node 必须显式输出主文档判断，不允许默认跳过。

输出格式（呈给[用户称呼]）：

```
主文档判断：[需要更新 / 不需要更新]
若需要：拟更新节 = §X，拟加内容 = ...（具体到节级 + 内容要点）
若不需要：理由 = ...（必须给出，不能写"无"或"协议层执行"等空泛词）
```

**执行规则**：

- "需要更新" → 等[用户称呼] C verdict 后才动主文档；[AI 名字占位]自己写（研究性主文档不外包）。
- "不需要更新" → 等[用户称呼] C verdict 后才进入步骤 3；[用户称呼]可纠正"其实需要"。
- 输出后静默等 verdict，不假设默认值。
- **无人值守例外**（daily-review 告别场景兜底补跑本 skill 时）：判断照常输出但不阻塞等待——主文档不动，判断挂 `长期记忆.md §上次未决` 待下次 verdict，流程继续后续步骤。

**不允许的判断模式**（标准滑动陷阱）：

- "本节点是协议层执行 / 工程动作，不是论证更新" — 滑动：把"是否提供论据"换成"是否变更顶层架构"。
- "主文档当前没人在写 / 本节点没产生新结论" — 滑动：把"主文档是否需要追"换成"主文档当前状态"。
- "_overview / progress 已经覆盖" — 滑动：_overview 是状态指针、progress 是推论链，均不替代主文档的论证 / 案例 / 理论锚点扩展。

**判准（不分项目类型，统一用）**——本节点提供以下任一类内容 → 需要更新：

- 新论据 / 新案例 / 新数据点（验证或反驳主文档已有论点）
- 理论锚点扩展（主文档引用的理论框架被延展 / 修正）
- 新文献支撑（应进 §References 或对应理论节）
- 决策固化（影响主文档已有结论 / 边界 / 设计原则）
- 框架更新（顶层架构 / 关键定义变更）

任何一项符合 → "需要更新"。全不符合 → "不需要更新" + 给具体理由。

**大版本变更 → 必归档 + 重写主文档**（不是追加章节）。触发条件（任一）：顶层架构性变更 / 决策性反转 / 主文档 ≥3 周未跟进而系统已多轮演化 / 命名版本体系重整。操作：

1. 旧主文档整体移动到 `_archive/main_documents/`（不存在则创建）。
2. `_archive/main_documents/README.md` 追加归档清单（版本 / 日期 / 文件 / 原因 / 替代物）。
3. 新版由主 agent（[AI 名字占位]）独立起草；sub-agent 只做事务性辅助，论证 / 理论锚点 / 案例 / 文献整合不可外包。
4. 起草期主文档状态标"起草中"；权威源临时回退到归档旧版 + ITERATION_LOG 全段。
5. 完成后更新 `_overview.md §项目文件` 指针。

**版本号连锁约束**：项目主文档版本号与对外发布版本号是强依赖，大版本变更时同步升级到统一值。

### 步骤 3 · `_overview.md`

按项目结构更新：§当前状态（推进到哪、断点在哪）/ §待解问题（关闭已解决、追加新涌现）/ §架构演进表（旧→新追加）/ §管辖文件（新增文件补指针）。

### 步骤 4 · `_progress/`

调用 `write-progress` 追加推进节点或更新现有节点。检查段间转换标记齐全：L2 `**开出**` + `新问题`；L3 `> **问题承接**` + 节末 `→ 开出的新问题` / `→ 本节收束`；段间转向逻辑点标 `→ 可提取转向模式`。标记定义权威源 = write-progress §2.3 追加与扩展；标记供 weekly-review 提取，不代表直接写记忆池。

## 阶段 B · 记忆的归档（步骤 5 · 节点级直升）

**本步标准（执行前先读）**：`00.memory_agent.md §身份层前置过滤 / §两轴判定 / §L0→L1 升格抽象红线 / §写入授权`。

输入限本次闭合节点。执行顺序：

1. **身份层前置过滤** → `00.memory_agent.md §身份层前置过滤`。
2. **分流落点**：

| 类型 | 落点 |
|---|---|
| 项目结论 / 具体内容 | 主文档 / progress，不进记忆池 |
| 段间转换判断逻辑 | progress 权威源 + `→ 可提取转向模式` 标记，周日梦提取，不直接升 |
| 节点内 ≥2 独立事件且过红线 | **当场直升** `episodic_memory.md` 活动态（N 级）→ `00.memory_agent.md §L0→L1 升格抽象红线` |
| 单次校准信号 | **零动作**——信号在会话转写中，当晚 dream 回放提取 |

3. **MEMORY_LOG 流水**（仅发生直升时）：主 agent 定稿一行 `YYYY-MM-DD | close-node：+N 条 episodic 直升；节点名`（日期 = 逻辑日期 → `daily-review/SKILL.md §逻辑日期判定`）→ **派storage-agent**追加 MEMORY_LOG `## 操作日志` 节末尾（格式护栏 → `storage-agent.md §log 写入格式护栏`）。

写入约束：`episodic_memory.md` 只写 1-3 星不启动注入；`semantic_memory.md` 一律不碰（快轨升格也归 dream）。

## 阶段 A 收尾

### 步骤 6 · 加载链 / 指针

扫 touched 文件：frontmatter / `## 加载链（上下游）` / 跨文件指针。新建文件 → 调用 `new-file` 补加载链；失效指针修正；新关联补同级联动。

### 步骤 7 · `_本周.md §任务清单`

本节点覆盖的 checkbox 勾为 `[x]`；新涌现任务追加 `[ ]`；过时描述改写或标注。

### 步骤 8 · 悬置与交接

悬置点 / 未解议题 / 待续讨论 → `_本周.md` 或 `长期记忆.md §当前处境 §上次未决`。close-node 只做节点级提取和指针，不做候选池代谢、衰减、毕业。

## 不做

- 不把 progress 当记忆池；不做项目推进到语义层的直通写入。
- 不写 `semantic_memory.md`；不做候选代谢、衰减、毕业（全归 dream）。
- 不触发身份层修改；不动 hook / settings。

## 执行等级

- 触发提议：N。
- 项目内常规同步（_overview / progress / _本周）：S/N，按项目权限与既有规则。
- 步骤 2 主文档判断输出本身：N（必须输出，等 verdict）；写主文档：C。
- 跨项目影响 / 覆盖既有主文档结论 / 身份层相关：C。
- 删除整份记忆文件：C；episodic 直升：N（→ `00.memory_agent.md §写入授权`）。

---

## 不允许外包

研究性内容（论证 / 理论锚点 / 案例分析 / 文献整合）由主会话独立撰写，不外包给事务性 sub-agent。事务性 sub-agent 可承担：文件归档 / 命名规整 / 多用户重组 / public repo 推送 / 代码层调试 / 事务性文档草拟——即重 IO、低判断密度的动作；任何涉及论证质量与研究判断的写入不外包。
