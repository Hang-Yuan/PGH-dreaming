# weekly-review

周日周复盘。两段：**A 在场段**（[用户称呼] 在线，分钟级，第 1→5 步）按序跑完才回应道别；**B 周日梦载荷**（本 skill 定义、当晚 dream 第 6 步执行）。记忆代谢判据本体全在 `<ASSISTANT_ROOT>\MEMORY\00.memory_agent.md`，本 skill 只走指针。

## 总纪律

- **在场段零记忆池写入**：episodic / semantic 全部代谢归当晚梦（`00.memory_agent.md §核心边界`「白天零记忆写入」）；在场段只动工作库（`_本周.md` / `<ASSISTANT_ROOT>\长期记忆.md`）与取 [用户称呼] 确认。
- **两 log 与长期记忆归档对象一律派 storage-agent**（`§写入授权`）：主 agent 定稿文本 + 派单，不直接 Read/Edit。
- **手动触发例外**：[用户称呼] 手动触发时，在场段 + 载荷可当场全跑（dream 无人值守语义不适用），C 级动作当场取 verdict。

## 输入
- `<ASSISTANT_ROOT>\00 专注区\_本周.md`（周次 / 任务清单 / 进展记录）。
- `<ASSISTANT_ROOT>\长期记忆.md`（§当前处境 / §时间轴 / §详细周录）。
- 记忆代谢规则权威源 `00.memory_agent.md`。

## 产物
- 在场段：周级全景报告 + 主线对齐更新（如有）+ 季度归档授权决定。
- 周日梦载荷交接（含载荷清单 + 季度归档授权标记），供当晚 dream 第 6 步读取执行。

# A 在场段（[用户称呼] 在线 · 第 1→5 步）

## 第 1 步 · 逻辑日期判定

按 `00.memory_agent.md §逻辑日期` 定周次 / 归档 / log / 长期记忆日期口径（物理 hour <06:00 算前一日）。取值先读实时物理时间再换算，禁脑内估算。

> v5.7.0 起逻辑日期权威源为 memory_agent，非 daily-review；周次判定统一至此。

## 第 2 步 · 全景摘要

从 `_本周.md` 生成本周全景：完成任务 / 进行中 / 遗留未完成项 / 实质产出 / 关键决策。

**遗留项三维过滤**（判断节点，防机械搬运上周 TODO 滚雪球）：每条未完成项过三维，命中任一即出局、不进"遗留"：
- ① 已被本周新事件收编（被后续工作覆盖）。
- ② 对当前主 / 次矛盾已无支撑（失去价值）。
- ③ 事务 / 调研型、可交托他人或他端。
三维全不命中才算真遗留，进全景的"遗留未完成项"。

## 第 3 步 · 给 [用户称呼] 补充空间

报完全景，问 [用户称呼] 有无补充 / 修正 / 本周特殊情况。无响应或道别级联 → 跳过，不阻塞。

## 第 4 步 · 判断节点 · 主线对齐

- **判断标准**：本周工作是否使 `长期记忆.md §当前处境` 的主线快照发生实质变化（主 / 次矛盾转移、阶段切换、新阻塞）。
- **触发钩子**：对照 §当前处境，有实质变化。
- **路由**：
  - 有实质变化 → 覆盖更新 §当前处境（派 storage-agent，N 级）。
  - 无 → 跳过，不为更新而更新。

## 第 5 步 · 判断节点 · 季度归档判定与确认（C 级）

- **判断标准**：是否到季度归档点——双触发，任一成立即触发：
  - (a) **自然季度边界**：本周日是本季度最后一个周日——本周日日期落在本季度三个月内（Q1 = 1 ~ 3 月 / Q2 = 4 ~ 6 月 / Q3 = 7 ~ 9 月 / Q4 = 10 ~ 12 月），**且** 本周日 +7 天已跨入下一季度。（纯日历归属判定，不依赖"周起于周一还是周日"，避免季末跨周歧义。）
  - (b) **满 13 周兜底**：`长期记忆.md §详细周录` 自上次季度归档以来周节数 ≥ 13 仍未归档（防自然边界被漏）。
- **触发钩子**：(a) 或 (b) 为真。(a) 用 `Bash date` 取本周日地面日期，判其月份属哪季 + 加 7 天判是否跨季；(b) 数 `长期记忆.md §详细周录` 的周节数。两者均禁脑内估算。
- **路由**：
  - 触发 且 在场取得 [用户称呼] 明确确认 → 把「季度归档已授权」写进第 6 步周日梦载荷交接，季度归档执行下沉到载荷 i。
  - 触发 但 取不到确认（[用户称呼] 未应 / 不在场）→ **本次跳过，不写授权标记**；下周仍成立时再提（`§写入授权` C 级不可被流程自动触发代替）。
  - 未触发 → 不提。

## 在场段收尾

第 1→5 步全部落地后才回应道别。产出周日梦载荷交接（下节），写入 `_本周.md` 末尾的临时交接节 `### 周日梦载荷交接`（工作库写入，S 级），供当晚 dream 读取。

# B 周日梦载荷（本节是载荷内容权威定义 · 当晚 dream 第 6 步执行）

dream 第 6 步带「周日梦载荷」标记时按本清单执行；每项开工先读所引 `00.memory_agent.md` 节，全部 N 级（做完 MEMORY_LOG 披露），毕业候选 C 级只生成不写。

**载荷项（按序）**：

- **a 账实核对**（先于 b-i 全部代谢 · 周级代谢必须建在完整账上）：
  - 账 = `_本周.md` 进展流水；实 = 文件修改痕迹。
  - 从进展记录「关联：」字段识别活跃项目与已记文件 → 扫各活跃项目本周修改的 `.md`，**排除 `_archive/` 等非产出目录**。
  - 差集（改了但流水没记）逐个**读前 30 行**判断（防只看文件名误判）→ 确属漏账按修改日期补记（S）；**日期拿不准标「待确认」并 MEMORY_LOG 记一笔**（防伪造时间）。
  - 任务清单有产出支撑却未勾的，补勾 `[x]`。
- **b 容量扫描 + 候选池重扫**（`§两池容量阈值`）：扫两池报数；升格候选池**每周清空不留库存**（过 `§升格判准` → 升 semantic 活跃组；不过 → 退 episodic 或删）。
- **c 横向统合**（`§横向统合`）：episodic 全表按深层结构聚类析母结构；**先固化后删、防 over-merge**。
- **d episodic 审查与衰减**（`§episodic 衰减` + `§升星`；周日梦是 D 类衰减唯一执行者）：容量红区**必须清到 ≤40**，清不到位 MEMORY_LOG 报剩余 + 原因；升星兜底补记（同一校准事件只计一次）。
- **e semantic 审查**（`§semantic 衰减` + `§项目语境快轨`）：命中补证据 / 升星；击穿标修订 / 降级；活跃组降级回 episodic 的去向按条目类型分流（跨情景 vs 项目语境不同）。
- **f 毕业候选生成**（`§毕业流程` + `§三路独立蒸馏`，hook 候选走 `§实测验证` 沙盒红线）：**只生成候选（N）挂 `长期记忆.md §当前处境` 上次未决等 C verdict，绝不写身份层**。
- **g 周总结落盘 + 周归档**（严格顺序，防丢内容 / 防下周无文件）：
  1. `_本周.md` 追加 `### 本周产出`（阶段全景 + 代谢统计，不一笔带过）。
  2. `_本周.md` 归档进 `<ASSISTANT_ROOT>\00 专注区\_归档\`，归档文件按 `YYYY-Wnn` 命名（年 + 周序号，如 2026-W26）。
  3. 更新 `长期记忆.md §时间轴` + `§详细周录`（派 storage-agent）。
  4. 按 `new-file` 规范创建新周 `_本周.md`。
- **h MEMORY_LOG 周复盘条目**（派 storage-agent，`§MEMORY_LOG 收录边界`「周复盘仅周日梦写」）：单周 15-25 行内，只记记忆系统自身代谢；不把 MEMORY_LOG 当新信号入口二次代谢。
- **i 季度归档**（**仅第 5 步已写「季度归档已授权」标记时执行**）：按 `storage-agent.md` 归档 SOP；**Bash 地面数字 + 新实例读回双校验**，确认落实才标完成（防未落实就标完成）。

> 载荷 a 是全部代谢的前置闸——账不全则后续容量 / 统合 / 衰减全部建在错误事实上。
> 通用日级代谢（衰减 / 统合 / 审查 / 候选 / 毕业的执行编排）见 dream 第 6 步；本节定义周级特有项（账实核对 / 周归档 / 周复盘 log / 季度归档）+ 在场段交接。

## 边界
- 在场段不碰记忆池、不写 MEMORY_LOG（代谢与流水都是当晚梦的账）。
- 不直接 Read/Edit 两 log 与长期记忆归档对象（一律派 storage-agent）。
- 周日梦载荷的**执行编排**（顺序 / 先固化后删 / N 级授权）由 dream 第 6 步落；本 skill 只定义载荷**内容**与在场段交接。
- 不自动执行季度归档（C 级，须在场确认；确认状态经周日梦载荷交接传给 dream 载荷 i）。
