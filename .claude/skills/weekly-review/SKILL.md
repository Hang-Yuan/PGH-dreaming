## 在场段流程（周日道别时，压缩到分钟级）

### 步骤 1 · 逻辑日期判定

按 `daily-review/SKILL.md §逻辑日期判定` 口径（权威源）。周次归属、归档、log 日期、长期记忆周录均用逻辑日期。

### 步骤 2 · 全景摘要

按 `_本周.md` 进展记录生成本周全景：任务完成情况 / 核心推进 / 关键产出 / 关键决策 / 跨项目连接 / 意外收获 / 遗留未完成项。

**遗留未完成项三维过滤**（必须执行，不可照搬旧任务清单）：

1. **被新事件收编** → 标"已收编"，不再列。
2. **已不重要**（对当前主要 / 次要矛盾无支撑）→ 标"已弃"，不再列。
3. **可交托**（事务性 / 调研型）→ 标"交托 [对象]"，入交托清单。
4. 过滤后仍存活的才进"遗留未完成项"。

### 步骤 3 · 给用户空间

全景摘要后自然询问：「这是本周的全景。有没有什么特别重要的事情、转变、或者你自己的感受，你想先说说的？」[用户称呼]补充纳入后续；无人响应（自动级联）→ 跳过。

### 步骤 4 · 主线对齐检查

对照 `长期记忆.md §当前处境`：本周是否推进了主要矛盾 / 偏移到主线外值不值 / §当前处境是否需要更新。有实质变化 → 覆盖更新（S 级）；无 → 跳过。

### 步骤 5 · 季度归档判定与确认（C 级）

**双触发判定**：(a) 自然季度边界——本周日 ≥ 该季度最后一个周日（Q1 含 3-31 / Q2 含 6-30 / Q3 含 9-30 / Q4 含 12-31）→ `quarter_end`；(b) 满 13 周兜底——派storage-agent数 `长期记忆.md §详细周录` 周节数 ≥ 13 仍未归档 → `overdue`。

任一为真 → **在场向[用户称呼]明示触发原因 + 归档对象（长期记忆 + 两 log），取得确认后**写入当晚梦载荷；无人在场取不到确认 → 本次跳过，下周仍成立时再提。skill 触发本身不构成授权。

> 在场段到此结束。后续全部进当晚梦（由 daily-review §排梦 带「周日载荷」标记交棒）。

## 周日梦载荷（dream 步骤 4 执行；每项开工前先读所引内核节）

**a · 账实核对**（账 = `_本周` 流水 / 实 = 文件修改痕迹）：从「关联：」字段识别活跃项目与已记文件；扫活跃项目本周修改的 `.md`（排除 `_archive/` 等非产出）；差集逐个读前 30 行判断，确属漏账 → 按修改日期补记 `_本周.md §进展记录`（S 级；日期拿不准标「待确认」记 MEMORY_LOG dream 条目）；任务清单有支撑未勾的勾 `[x]`。补账完成才进后续载荷——周级代谢必须建立在完整的账上。

**b · 容量扫描 + 候选池重扫**：两池报数（→ `00.memory_agent.md §两池容量阈值`）；§升格候选池逐条过 `00.memory_agent.md §升格判准`——过 → 升启动注入组（N 级）；不过 → 退回 episodic 或删。候选池每周清空，不留库存。

**c · 全表横向统合**：→ `00.memory_agent.md §横向统合`（活动+复审全表聚类 → 析母结构 → 先固化后删除 → 防 over-merge）。

**d · episodic 审查与衰减**：→ `00.memory_agent.md §episodic 衰减 / §升星 / §有效工作日`。逐条审查活动 / 复审条目；本周被引用 / 提取 → 回写最后激活；衰减状态机执行（周日梦是 D 类唯一执行者）；升星兜底补记（同一校准事件只计一次）；容量红区 → 必须清到回 40 以下，清不到位 MEMORY_LOG dream 条目报告剩余与原因。

**e · semantic 审查**：启动注入组逐条——本周命中 → archive 补证据、必要时升星；击穿 / 失效 → 修订、降级、退出注入（→ `00.memory_agent.md §semantic 衰减`；项目语境条目降级走休眠态）；相似合并；项目语境条目跨项目命中 → 摘标转正（→ `00.memory_agent.md §项目语境快轨`）；达 6 星 → 标毕业候选。

**f · 毕业候选生成**：按 `00.memory_agent.md §毕业` 完整流程执行（判型分流 → 三路独立蒸馏 → 收敛判定 → hook 实测验证 → 毕业提案格式输出）。只生成候选（N 级）记 MEMORY_LOG dream 条目 + 挂 长期记忆.md §当前处境 上次未决等 C verdict，不写身份层。

**g · 本周总结落盘与周归档**：`_本周.md §进展记录` 末尾追加 `### 本周产出`（阶段全景 + 代谢统计，不可一笔带过）→ `_本周.md` 归档为 `_归档/YYYY-Wnn.md` → 更新 `长期记忆.md §时间轴` + `§详细周录` 追加 → 创建新周 `_本周.md`（加载链按 `new-file` 规范）。

**h · MEMORY_LOG 周复盘条目**：按下方模板定稿 → 派storage-agent追加 `## 周复盘` 节末尾。只记记忆系统自身代谢，单周 15-25 行内。

**i · 季度归档**（仅在场段步骤 5 已取得确认时）：派storage-agent按 `storage-agent.md` SOP 执行（两 log §归档 SOP / 长期记忆 §长期记忆归档变体）；核查按 `storage-agent.md §归档核查协议`（Bash 地面数字 + 新实例读回）；结果记 MEMORY_LOG dream 条目。

## MEMORY_LOG 周复盘条目

模板（本节为周复盘条目模板唯一权威源）：

```markdown
### YYYY-WNN 周复盘（YYYY-MM-DD）

**本周记忆操作**：
- 回放覆盖：[本周 dream 跑了几晚 / 缺勤补扫情况]
- 情景层新增 / 合并：[条目 ★星；含候补转正与唤醒]
- 情景层状态变化：[活动 / 复审 / 候补 / 休眠 / 已物理删除]
- 语义层新增 / 合并：[条目名称 ★星；含项目语境快轨与摘标转正]
- 升格候选池：[提名 N / 升格 M / 退回 K]
- 横向统合：[母结构 ← 原料条目列表]
- 修正 / 删除 / 毕业候选：[条目及去向 / 原因]

**强度变化**：[情景层 / 语义层升降]

**衰减处理**：[处理了哪些条目，或「无」]

**复盘发现**：[仅记忆系统层面的问题 / 模式 / 改进点]

**下周迭代行动**：[仅记忆系统层面的调整]

**参数调整**：[阈值 / 判准 / 节律参数变动，或「无」]
```

## 不做

- 在场段不做记忆池操作（全部归周日梦载荷）。
- 不执行自动毕业（候选挂 长期记忆.md §当前处境 上次未决等 C verdict）；不把 MEMORY_LOG 当新信号入口。
- 不让升格候选池留过周库存。
- 不直接 Read/Edit 两 log 与长期记忆归档对象（一律派storage-agent）。
- 不动 hook / settings。
