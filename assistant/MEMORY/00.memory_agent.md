## 总则

### 单一权威源声明

本文件是记忆代谢规则的**唯一权威源**。以下规则只活在本文件，其他位置（skill、记忆文件头部、CLAUDE.md）一律只写指针，不复制规则本体（清单与本文件节名逐字一致，可直接检索）：

### 核心边界

记忆系统只记录会改变未来预测或行动的信号。事实本身不构成写入理由。

**白天零记忆写入**：工作会话中不做任何记忆池操作——校准信号以完美保真度活在上下文与会话转写中，夜间由 dream 统一提取。

**白天零记忆注入**（v5.6.0）：inbox 退役后，白天不再需要常驻 schema 做实时判断——运行时的共同世界模型底座 = **USER + SOUL + CLAUDE.md §R 三件套身份层**（启动注入），semantic 池退到幕后，白天对运行时不可见。semantic 只在夜间被 dream 读取代谢。

两个记忆池：

| 文件 | 启动注入 | 职责 | 容量 |
|---|---|---|---|
| `episodic_memory.md` | 否 | 1-3 星情景级候选 schema（活动 / 复审 / 候补 / 休眠） | 活动+复审 ≤30 健康（→ §两池容量阈值） |
| `semantic_memory.md` | **否**（v5.6.0 起退出注入） | 4-6 星语义级 schema 的中间工作区，等毕业进身份层 | 活跃组 10 硬上限 |

身份层（= 白天唯一注入的共同世界模型底座）：

| 位置 | 职责 |
|---|---|
| `USER\USER.md` | 用户长期稳定特质。 |
| `SOUL\persona\persona_SOUL.md` | AI 长期稳定行为模式。 |
| `~/.claude/skills/` | 流程化、可执行的稳定行动规则。 |
| `CLAUDE.md §R` | 全局、需模型判断、高频的思考纪律。 |

默认动作是遗忘。只有被预测关系和用户校准信号反复支持的模式，才允许上行；稳定到终态的 schema 毕业进身份层（白天注入的唯一出口）。

## 记忆系统结构

> 本节是记忆系统的**结构性全景描述**——回答"这个系统由什么构成、信息怎么流动、各层拿什么和谁比对、最终去哪"。本节只描述机制是什么（what）与为什么（why），**不规定任何执行者按什么顺序敲哪些动作**（how 归各 skill：dream / close-node / review skill）。
>
> 本节须自足到——一个零上下文的 agent 仅凭本节就能推导出"代谢该和什么比对、信息该流向哪里"。

### schema 是什么

记忆系统流转的基本单位是 **schema**，不是"事实"。一条 schema = 一个**预测单元**：

```
触发情境（何时调用这条 schema）  →  行动 / 预期（该怎么预测或行动）
```

- **本体两件**：触发情境 + 行动/预期。程序型必填"行动"，语义型必填"预期"。
- **附带元数据**：强度（★ 数 = 稳定性主张）/ 状态 / 类型（程序·语义·混合）/ 语境（跨情景·项目名）/ 证据指针。
- **成立判据**：能画出一个"不这样做的真实对照人"才算 schema；画不出 = 塌成全称重言式，不是 schema（→ §升格判准 对照人测试）。

事实本身不是 schema——只有"会改变未来预测或行动"的模式才进入系统（→ §核心边界）。**这一点是后面所有差分比对的前提**：比对的是 schema（预测单元）跟 schema，不是事实跟事实。

### 信息流与分流

唯一原始入口是 **L0 = 会话转写**（当日全部对话的完美保真记录）。从对话到身份层的完整流动——注意两处关键结构：**① episodic 有两个提取来源**（对话 + progress 节点转向）；**② 工作库与记忆系统在 progress 处交叉**（progress 物理上属工作库，其"节点怎么转、为什么转"反向喂进记忆提取管线）。

```
对话转写（L0 · 唯一原始源）
   ├── 事件走向（做了什么/选了什么/事实层） → 工作库（项目主文档 · 长期记忆 · _本周）
   │                                              └── _progress（节点转向：怎么转·为何转）
   │                                                    ├─ 提取来源②：progress 节点转向 ┐
   │                                                    └─ 快轨：项目语境 ≥3 工作日复现直升 semantic
   └── schema 走向（做了什么判断+为什么/校准） ── 提取来源①：对话 ┐
                                                                  ▼
                            两轴判定（对照现有池做差分 → §两轴判定）
                                       │  提取 + 抽象（≥2 独立事件 → 活动态）
                                       ▼
                            episodic_memory（1-3★ · 情景级候选层 · 升星/降星/衰减，默认遗忘）
                                       │  跨情景 ★★★ 再命中 → 周日横向统合（升 semantic 唯一门）
                                       ▼
                            semantic_memory（4-6★ · 语义级 schema 中间工作区 · 等毕业）
                                       │  6★ + 跨周稳定 ≥3 周 + [用户称呼] C verdict
                            最后分流 ──┴── 语义型 → USER / SOUL（终态身份层）
                                          程序型 → [用户称呼]授权 → 启用 meta-skill → 蒸馏产出新 skill
```

> **图为主干**：程序型毕业出口此处只画 skill 一条（经 meta-skill 通道）；hook / §R 等细分出口仍在 §毕业 决策树内，全景图不穷举。

### 代谢对照基线（差分的减数）

**两轴判定本质是差分运算，不是孤立打分**——这是上面流程图中央"两轴判定"框最容易被漏掉的前提，也是本节存在的核心理由。

P轴问的是"这条新观察与**当时已有 schema 的预测**是什么关系"（命中 / 击穿 / 未覆盖）；升星、查重、合并、唤醒同样都是相对现有条目的动作。**差分必须有减数**：没有"现有 schema 全集"同时在场，就判不出"命中已有"还是"全新"，只能凭模型默认世界知识瞎猜——结果是该升星的重复新建、该合并的各自落盘、该击穿的漏判。

因此代谢的对照基线 = **现有 schema 全集**，三层各自的比对对象与角色：

| 基线层 | 拿它比对什么 | 角色 / 在场理由 |
|---|---|---|
| `episodic_memory.md`（1-3★ 中间态） | 新观察是否命中已有候选 → 升星；是否同构 → 查重合并；是否命中休眠 → 唤醒 | 中间态 schema 的爬星就靠这层比对 |
| `semantic_memory.md`（4-6★ 中间态） | 新观察是否命中跨情景 schema → 强化（爬星燃料）；是否击穿 → 标修订 | **v5.6.0 退注入后，代谢期是 semantic 唯一被读取的时机**——白天它不进运行时上下文，作为对照基线的角色只在代谢期成立。缺了它 semantic 等于既不注入也不被读、彻底悬空 |
| 身份层 `USER` / `SOUL` / `§R`（终态） | 新观察命中终态 → 不加权（已毕业不重复强化）；抵触/击穿 → 才是身份层待更新信号 | 启动序列已注入，代谢期天然在场，无需额外读取 |

比对后"命中谁 → 怎么动"的分流规则在 §身份层前置过滤 + §提取过滤下限：核心是**先看命中的是哪一层**——命中终态（USER/SOUL）不提取，命中中间态（episodic/semantic）提取强化，差别在这一层是否还在路上。

> **基线取主文件**：episodic / semantic 主文件本就设计成极简（证据详记在 `_archive/`），比对深层结构同一性靠主文件即可，archive 不进基线。**skill 不进基线**——它是已毕业的程序型终态，由 meta-skill 通道独立审改演化，不参与记忆池的差分代谢。

### 生命周期（候选 → 终态）

一条 schema 从对话到身份层的完整上行，每段节律见 §代谢速度梯度；默认动作是遗忘，绝大多数候选在 episodic 衰减出局：

```
L0 对话转写 / progress 节点转向
   │  两轴判定（对照基线做差分）              → §两轴判定 · §身份层前置过滤
   ▼  提取 + 抽象（≥2 独立事件跨情景 → 活动态）  → §L0→L1 升格抽象红线
episodic_memory（1-3★）
   │  命中升星 / 无激活转复审 → 物理删          → §升星 · §episodic衰减
   ▼  跨情景 ★★★ 再命中 → 周日横向统合（唯一门）   → §升星 · §横向统合
semantic_memory（4-6★）
   │  命中强化 / 击穿修订 / 衰减 / 横向统合      → §semantic衰减
   ▼  6★ + 跨周稳定 ≥3 周 + [用户称呼] C verdict  → §毕业
身份层 USER / SOUL / skill / §R（终态 · 白天注入的共同世界模型底座）
```

越往上代谢越慢（稳定性主张越强）；到身份层即毕业、退出代谢、成为白天注入底座，不再回到记忆池差分。

## 全局参数

### 代谢速度梯度

代谢速度与稳定性主张成反比——越往上越慢，是设计不是缺陷：

| 层 | 节律 | 执行者 |
|---|---|---|
| 工作结论 → 工作库 | 小时级（节点闭合即固化） | close-node 阶段 A / daily-review 阶段 A |
| 行为 schema → episodic | 日级 | dream（夜间回放） |
| 跨情景 schema → semantic | 日级快轨（项目语境）/ 周级慢轨 | dream（日级 + 周日载荷） |
| 身份层 | 月级 | 毕业（[用户称呼] C verdict） |

短期记忆 = 会话上下文本身，不设代谢机制。

### 有效工作日

衰减时钟的单源定义。

**定义**：当日 `_本周.md §进展记录` 有流水追加 = 1 个有效工作日。

- **回推方法**：数当周（必要时含归档周文件）有工作段的日子。
- 纯对话、无工作流水的日子不计入——衰减时钟按"系统被实质使用的日子"走。

### 逻辑日期

记忆代谢全局时间口径的单源定义。所有 skill 的日期归属、周次判定、log 日期、探针日期统一至此（daily-review / weekly-review / close-node / dream 指针至此，不在各 skill 内重定义）。

**定义**：物理 hour < 06:00 → 逻辑日期 = 物理日期 − 1；≥ 06:00 → 同物理日期。

- **用途**：`_本周` 流水日期 / daily·weekly·close-node·dream 的日期字段与周次判定 / `last_dream.md` 探针。
- **cron 排程字段不适用**：cron 按物理钟触发，排程时刻一律填物理时间；凌晨跨日时严禁把逻辑日期填进 cron 日期字段，逻辑日期只写进 prompt 正文。

### 两池容量阈值

| 池 | 健康区 | 黄区（预警） | 红区（强制清） | 硬上限 |
|---|---|---|---|---|
| `episodic_memory.md` 活动+复审 | ≤ 30 | 30-40（提示衰减扫描） | > 40（周日梦必须清到回 40 以下 → weekly-review §周日梦载荷） | — |
| `episodic_memory.md` 候补 | ≤ 10（软提示线，超出 = 信号洪水，dream 在 MEMORY_LOG 条目提示） | — | — | 7 有效工作日自删，无需硬限 |
| `episodic_memory.md` 休眠 | 不计容量、不占活动池 | — | — | 季度审查出清 |
| `semantic_memory.md` 活跃组 | ≤ 5 | 6-7（无动作） | 8（预警 + 可退最低优先项到备用组） | 10（必须强退最低优先项回 episodic） |
| 活跃组内项目语境条目 | ≤ 3（硬限，超出退最低优先项回 episodic 休眠态） | — | — | — |

**容量扫描强制步骤**：dream 开工必扫两池条数 → 报数 + 黄区提示 / 红区触发当次清理；结果写入 MEMORY_LOG dream 条目。

## 提取层

对话转写（L0）→ 记忆池的入口层：先按输入规范回放，再逐条过两轴判定。

### L0 回放提取规则

dream 的输入规范。L0 = 会话转写，不是记忆文件——完美保真、零维护、永远在场。

1. **定位**：glob `<CLAUDE_HOME>\projects\` 下修改时间属当日（逻辑日期）的 jsonl（覆盖全部会话）。
2. **分段回放**：派storage-agent按块读取 → 返回候选信号清单（带两轴预标 + 原文锚点）；主循环逐条按 §两轴判定 裁决。重 IO 派子 agent 可指定低成本模型；裁决留主循环。
3. **回放深度**：重日全量、轻日只扫尾段；以当日 `_本周` 流水段数自判。
4. **补扫上限**：缺梦补扫最多回放最近 **3 个有效工作日**的转写；更早的接受信号损失（默认遗忘）。
5. **校验职责**：回放时核对事件走向是否已被阶段 A 固化进工作库——漏账不补写，MEMORY_LOG dream 条目报漏（工作库写入是在场段职责）。

#### 双走向分流

- **事件走向（主支）**：当日做了什么 / 选了什么 / 推到哪 → **在场段**（close-node / daily-review 阶段 A）固化进工作库（项目主文档 / `_progress` / 长期记忆 / `_本周`）。梦不重做，只校验。
- **schema 走向（支流）**：做了什么判断 + 为什么 / 用户校准 → **dream** 按本节提取、按下方各节代谢。
- 硬顺序天然满足：梦在道别（阶段 A 完成）之后才被排定。

### 两轴判定

dream 回放提取的核心判准。每条候选信号判两个问题：

1. **P轴：预测关系**。这条观察与当时 schema 预测是什么关系？
2. **C轴：校准信号**。[用户称呼]是否给出了明确校准？

#### P轴

| 值 | 定义 |
|---|---|
| 命中 | 观察符合当时 schema 的预测。 |
| 击穿 | 观察反向校准或否定当时 schema 的预测。 |
| 未覆盖 | 当时 schema 没有覆盖这类观察。 |

#### C轴

| 值 | 定义 |
|---|---|
| 正向确认 | 明确赞同、确认、满意或强化。 |
| 反对 | 明确反对、不满或拒绝。 |
| 纠正 | 修正某点，但非全盘否定。 |
| 中性 | 没有给出明确校准。 |

#### 提取过滤下限

```text
P轴=命中 且 C轴=中性 -> 不提取
其他组合 -> 进入代谢判定
```

`命中 + 正向确认` 要提取；它是稳定 schema 的强化证据。

#### 身份层前置过滤

提取 / 升格前先匹配，**关键是先看命中的是哪一层**——终态层（USER/SOUL）不被日常证据加权，中间态层（semantic 活跃组）的命中正是升星燃料：

| 命中对象 | C轴 | 动作 | 理由 |
|---|---|---|---|
| **USER / SOUL**（终态身份层） | 中性 **或 正向确认** | **不提取** | 已毕业终态，符合旧认知不重复加权 |
| **USER / SOUL**（终态身份层） | 抵触 / 新扰动 / 击穿 | 进代谢判定，**不直接改身份层** | 这才是身份层需更新的信号 |
| **semantic 活跃组**（中间态） | 正向确认 / 命中 | **提取**（archive 补证据、按 §升星 加权） | 未毕业 schema 的强化证据 = 爬星燃料 |
| **semantic 活跃组**（中间态） | 击穿 | 标记修订（→ §semantic 衰减） | — |
| **未命中身份层 / semantic** | 任意 | 正常代谢判定（走 §提取过滤下限） | 全新信号 |

**判准核心**：同样是「命中 + 正向确认」，命中**终态**（USER/SOUL）不提取，命中**中间态**（semantic）要提取。差别在这一层是不是还在路上——终态再确认也不动，中间态靠确认爬向毕业。

## episodic_memory 规则

保存 1-3 星候选，程序型 / 语义型统一池，`类型` 字段区分。

### episodic 状态机

| 状态 | 含义 | 进入 | 退出 |
|---|---|---|---|
| 活动 | 正常候选 | ≥2 独立事件过红线升格 | 升 semantic / 衰减转复审 / 物理删 |
| 复审 | 衰减观察期 | 7 有效工作日无激活且提取 0 | 再激活回活动 / 14 日仍无激活物理删 |
| 候补 | 单证据等复现 | 单事件值得追踪（dream 判） | 第二事件到达 → 转活动；7 有效工作日无第二证据 → 物理删 |
| 休眠 | 项目语境条目蛰伏 | 曾达 semantic 的项目语境条目降级时 | 回放命中 → 唤醒回活动（保留星级）；季度审查无唤醒 → 物理删 |

休眠条目免时间衰减、不占活动容量、周日梦不扫描；唤醒走 §升星 的命中匹配机制。

### 生成（dream / close-node）

1. 按预测对象、校准信号、情境域聚合候选信号。
2. 判断是否可写成可复现的"情境 → 行动 / 预期"。
3. 过 §L0→L1 升格抽象红线：≥2 独立事件 → 活动态；单事件值得追踪 → 候补态 ★。
4. 与现有条目轻量查重：深层结构同一 → 升星不新建；疑似同构 → 标 `待统合簇` 留周日梦。

### L0→L1 升格抽象红线

- 形式达标（情境→行动/预期句式）+ **跨事件情境抽象**（≥ 2 个不同事件支撑同一类情境）两者同时满足 → 活动态。
- 单事件换句式不算抽象；同节点内同一事件多次提及不算 ≥2 独立事件。
- 单事件直升活动态仅限：高风险 / 越权 / 系统事故 / 重大安全信号；须写明复审条件。其余单事件 → 候补态。
- 自检：只看这一条单事件证据能不能写出这条 schema？——能 = 换句式 / 不能 = 跨事件抽象。

### 字段

| 字段 | 说明 |
|---|---|
| 强度 | ★ / ★★ / ★★★ |
| 状态 | 活动 / 复审 / 候补 / 休眠 |
| 类型 | 程序 / 语义 / 混合 |
| 语境 | 跨情景（默认，可省）/ 项目名（项目语境条目必填） |
| 创建日期 / 最后激活 | 日期 |
| 提取次数 | 被提取为 semantic 或用于复审的次数 |
| 触发情境 / 行动模式 / 预期 | schema 本体（程序型必填行动，语义型必填预期） |
| 来源指针 | 转写日期 + 会话 / progress / archive |

### 升星

| 条件 | 动作 |
|---|---|
| 同一日内再次校准 | 只计 1 次（升星上限 1 次/日） |
| 7 个有效工作日内再次被校准 | 强度 +1；更新最后激活 |
| 回放命中休眠条目 | 唤醒回活动态，保留原星级，更新最后激活 |
| 被明确击穿 | 重写情境 / 行动 / 预期，或新建竞争候选 |
| 强度达 ★★★ 且再被命中一次 | **不直升 semantic**——提取次数 +1、更新最后激活，转标 `待统合簇`（或维持 ★★★ 留 episodic）；该条留待周日横向统合判去向。★★★ 再命中 = 怀疑触发，不是升格触发（见下方说明） |

执行者：dream（当日回放时当场执行）；周日梦兜底补记。同一校准事件只计一次。

**★★★ 再命中 = 怀疑触发，不是升格触发**：跨情景条目爬到 ★★★ 顶格后再被命中，重点不是"够星了升上去"，而是"它凭什么单条就够格 = 该被怀疑是否是某未成形母结构的单一表面"——这个判断需要全表在场比对，日级 dream 的小上下文做不了。故 ★★★ 再命中只触发怀疑（标 `待统合簇`），把升格判断推到 **§横向统合（周日梦）= 跨情景升 semantic 的唯一门**。

**两条例外（日级直升，不经唯一门）**：
- **项目语境快轨**：同项目 ≥3 有效工作日复现的项目语境条目，按 §项目语境快轨 日级升（时效性优先）。
- **高风险 / 越权 / 系统事故单条**：§升格判准 第 2 条所列可单条保留者，日级直升即时生效（不能等一周）。

### episodic 衰减

活动 / 复审态走状态机（→ §episodic 状态机），不做渐进降星：

| 条件 | 动作 |
|---|---|
| 7 个有效工作日无激活且提取次数 0 | 活动 → 复审 |
| 14 个有效工作日仍无激活（已复审） | **物理删除** |
| 候补 7 个有效工作日无第二证据 | **物理删除** |
| 提取次数 1 且 14 个有效工作日无再激活 | 复审（提取过的耐心更长） |
| 提取次数 ≥ 2 | 不按时间自动删 |
| 提取次数 ≥ 5 | 复审是否应合并进 semantic 或项目规则 |
| 休眠态 | 免时间衰减；季度审查整批出清 |

执行者：周日梦（D 类衰减唯一执行者；候补到期删由每日 dream 顺手执行）。

## semantic_memory 规则

语义级 schema 的中间工作区（4-6 星，等毕业进身份层）；**不启动注入**，仅夜间被 dream 读取代谢。主文件极简；证据全部写入 `_archive/semantic_archive.md`。

### 升格判准

从 episodic / progress 提取 semantic 必须满足 1、2、4-8 条；第 3 条是**分流器**不是闸门：

1. 能写成非机械的「预测情境→行动预期」。
2. 跨日 / 跨节点 / 跨情境复现；重大安全 / 越权 / 系统事故可单条保留，注明复审条件。
3. **领域无关性分流**：抽掉项目名 / 理论名 / 具体对象，换一个不相干项目仍成立 → **跨情景 schema**（慢轨）；不成立但满足 §项目语境快轨 条件 → **项目语境 schema**（快轨，强制标注）；两者皆否 → 不升。
4. **对照人测试**：能画出一个常见、真实的「不这样做的对照人」。画不出 = 塌成全称重言式，不升 / 已升的降。
5. **per-entry 判**：每条单独过判准，不拿单一结论一刀切全表。
6. 有 P轴 / C轴证据，不是单纯事实正确。
7. 不与 USER / SOUL、项目主文档、长期记忆冲突。
8. **内在模式深入**：能抽象到比当下证据更深一层的内在稳定模式。形式达标但本条不达标 = 机械搬运，不升。

### 项目语境快轨

- **升格条件**：同一项目内 ≥3 个有效工作日复现（升星节律下自然 = 三天周期）+ 过判准 1、2、4-8 条 → 升 semantic，**强制「语境：项目名」标注**。
- **进活跃组**：进活跃组参与排序，组内项目语境条目硬限 3 条（→ §两池容量阈值）。
- **摘标转正**：项目语境条目积累出跨项目命中证据 → 摘除标注，转跨情景 schema（周日梦重扫时判）。
- **降级去向**：衰减降到 3 星时回写 episodic **休眠态**（区别于跨情景条目回活动态），等项目重启唤醒。

### 升格动作（N 级，dream 执行）

1. 检查是否已有等价 schema（同类触发给出同一预测、同一纠偏方向、同一行动约束）。
2. 已有 → 合并到既有条目，证据写 archive；无 → 新增条目，证据写 archive。
3. 执行顺序硬约束：本体 + 证据**先**落盘，确认后才物理删除贡献来源（先固化后删除）。
4. 容量分级动作 → §两池容量阈值。
5. 全部升格 / 退回 / 删除当晚 MEMORY_LOG dream 条目披露。

### 横向统合（周日梦执行，N 级）

**跨情景 episodic → semantic 升格的唯一门**（项目语境快轨 + 高风险单条两例外走 §升星 日级直升，不经此门）。日级 dream 不把跨情景 ★★★ 直升 semantic，只标 `待统合簇` 喂到这里——保证所有进 semantic 的跨情景条目都过了"与同级比对、防 over-merge"这道闸，semantic 不被单条碎片化。

1. **聚类**：按深层结构（不按表层措辞）把 episodic 全表（活动 + 复审；不含候补 / 休眠）分簇；优先纳入带 `待统合簇` 标记的条目。
2. **逐簇析母结构**：母结构必须过 §升格判准。
3. **统合事务**：母结构 + 证据先落 semantic / archive，确认落盘后才物理删除贡献原始条目；**防 over-merge**——再合并就画不出对照人 → 停止合并。
4. **单条升格**：不成簇的单条（含 `待统合簇` 中查无同簇者），过判准够格 → 直接升。

### 主文件字段

| 字段 | 说明 |
|---|---|
| 强度 | ★★★★ / ★★★★★ / ★★★★★★候选 |
| 状态 | 活跃组 / 备用 / 需要修订 / 毕业候选 |
| 类型 | 程序 / 语义 / 混合 |
| 语境 | 跨情景（默认，可省）/ 项目名 |
| 预测情境 / 行动预期 | schema 本体 |
| 证据指针 | `_archive/semantic_archive.md` 对应条目 |

### 排序

活跃组优先级（dream 代谢关注序，非启动注入）：高风险行为约束 > 强度更高 > 最近 14 天命中 > 与当前处境或本周任务相关 > 未被身份层稳定覆盖。项目语境条目在其项目活跃期内优先级上浮、静默期下沉。

### semantic 衰减

| 条件 | 动作 |
|---|---|
| 整周无命中 | 强度 -1 |
| 被击穿并纠正 | 标 `需要修订`；周日梦重写或降级 |
| 跨情景条目降到 3 星 | 移出活跃组，回写 `episodic_memory.md` 活动态 |
| 项目语境条目降到 3 星 | 移出活跃组，回写 `episodic_memory.md` **休眠态** |
| 活跃组达 8 条 | 预警 + 可退最低优先项到备用组 |
| 活跃组达 10 条 | 强制将最低优先项退回 episodic |
| 达到 6 星且跨周稳定 | 生成毕业候选，等待 C verdict |

## 毕业

### 毕业条件

```
6星 + 跨周稳定 >= 3 周 + [用户称呼] C verdict
```

高风险纪律可在 6 星 + 2 周稳定时提出候选，但仍需 C verdict。项目语境条目不毕业（先摘标转正才有毕业资格）。

### 判型分流

**语义型 → USER（唯一出口）**

- 毕业线 = 星级满 + 三路蒸馏收敛（见 §三路独立蒸馏）
- 蒸到根 = 挖世界观假设："为什么是这样不是那样" → 关于世界的命题；检验 = 反事实反转（假设反转，行为必须跟着反转）；停手线 = 直接支撑本机制的层，不追公共根
- 形态门槛：四件套写全（机制句 / 世界观假设 / 触发情景 / 预测动作），或切分判准查重判定为并入已有条目
- 时间判准把门：情境绑定的不进 USER

**程序型 → 四出口决策树**（按序判定，命中即止）

| 序 | 出口 | 判别 | 硬性约束 |
|---|---|---|---|
| 1 | hook | 触发可代码化、动作确定性 | 脚本 ≤30 行；单一判定 + 单一动作；超线 = 蒸馏未到位退池；必须经 §实测验证 |
| 2 | SOUL | AI 人格/关系层模式（纠错史可挖出假设） | 四件套格式 |
| 3 | skill | 绑定特定流程域 | 写入对应 skill 管辖节 |
| 4 | CLAUDE.md §R | 全局、需模型判断、高频 | 单条 ≤80 字；一次毕业至多 1 条；写不进 80 字 = 没蒸熟退池 |

**混合型**：语义面 / 程序面拆开各走各口，允许不同时毕业。

### 三路独立蒸馏（全出口通用）

1. 梦内对毕业候选判型定出口后，**并行拉 3 个 sub-agent**，彼此不可见，各自只拿：条目原文 + semantic_archive 该条全部证据 + 对应出口规范（含字数约束）
2. 各自独立产出蒸馏物（USER 口 = 四件套；hook 口 = 触发条件 + 拦截动作 + 脚本草稿；§R 口 = 80 字条文；SOUL/skill 口 = 对应格式）
3. 收敛判定：核心参数一致（hook 比较触发事件 / 条件逻辑 / 动作三参数；USER 比较假设命题的实质等价）
   - 三路一致 → 收敛，进下一道（hook 进 §实测验证，其他口进提案生成）
   - 两路一致 → 多数决，提案标注分歧点
   - 三路各异 → 未熟，退池继续攒，当次不生成提案

### 实测验证（hook 专属）

1. 取收敛版实现，存暂存区 `<ASSISTANT_ROOT>\MEMORY\_dream_staging\`（**全程不触 `settings.json`、不进 `~/.claude/hooks\` 生效目录**——安全红线）
2. 再并行拉 3 个 sub-agent，各自独立设计测试集（应触发 / 不应触发 / 边界 case），沙盒实跑脚本回报成功率
3. 三路全过（应触发全命中、不应触发零误伤）→ 提案升级"已验证"；任一路失败 → 失败 case 附卷退回

### 毕业提案固定格式

条目 ID / 判型与出口 / 蒸馏产物全文 / 三路收敛报告 / 实测报告（hook 专属）/ 建议落点与替换文本。

### 毕业流程

1. 周日梦只生成候选（N 级，依 §三路独立蒸馏 + §实测验证 执行）
2. MEMORY_LOG dream 条目留账 + 挂 长期记忆.md §当前处境 上次未决，week-sync 提醒待裁决
3. [用户称呼] C verdict 后才写正式目标
4. 写入前剥壳：去星级、去临时证据、去现场情绪、去重复来源，只保留稳定描述和必要边界
5. 写入后，`semantic_memory.md` 条目标记已毕业并移出活跃组
6. 完整审计 → semantic_archive + MEMORY_LOG §毕业归档（派storage-agent）

混合类型优先拆分；不能拆时只保留一个权威源，其他位置写指针。
