# 专注区 · 区域 agent

专注区是一个**按周切片的工作区**，不是一套要遵守的节律。它只做一件事：装当下在动的活，以及暂时无家可归的东西。东西在这里停留、流动，最终该进项目的进项目、该进素材库的进素材库——这里不留终态。

## 加载链（上下游）

**上游（由谁加载本文件）：**
`CLAUDE.md §I 系统原则` — 任务涉及专注区时，由区域规则触发按需加载。启动序列步骤 4 读取 `_本周.md`，步骤 5 调用 `week-sync`。

**管辖文件（下游）：**
- `_本周.md` — 当前周工作台（工作记忆，读写）
- `_归档/YYYY-Wnn.md` — 历史归档（只读）

**下游执行 skill：**
下列 skill 按本规范操作专注区；本规范更新后，由 meta-skill 或人工审查对齐各 skill 实现。
- `week-sync` — 会话启动同步；周一首会话提示孤儿文件 + 推理可能归属（只读建议，见 §孤儿文件机制）。
- `daily-review` — 在场段把当日工作结论固化进工作库（含 `_本周.md` 进展流水）。
- `weekly-review` 周日梦载荷 — 周归档 + 本周产出补写（当晚 dream 无人值守执行，见 §归档）。

- `02 阅读区/00.阅读区_agent.md` / `03 写作区/00.写作区_agent.md` — 阅读 / 写作任务也在 `_本周.md` 追踪。
- `MEMORY/00.memory_agent.md` — 校准信号留在会话转写里，由当晚 dream 代谢，不与 `_本周.md` 混合。

## 区域定义

**专注区 = 用户的周工作台。** 一周一个文件（`_本周.md`），边界清晰。它装两类东西：

1. **当周在动的项目活跃面**：项目主文档在项目区，这里只放它这周在动的那一摊（流水进展 + 任务）。
2. **无家可归的东西**：不属于任何具体项目、临时聊出来的零散想法或文件。它们没有别的地方去，就落在工作区里，直到找到家。

性质是**短期、在制品、会流动出去的**：原始想法在这里变成结构化产出，决策在这里浮现和追踪，在制品在这里停留直到准备好进入项目或素材库。**它是中转站，不是归宿**。

> 专注区不规定"周一规划、周末收尾"这类工作节律。每项工作按自己的生命周期走——有的三小时干完，有的跨好几周，急活随时插入。"周"只是工作台换纸与回顾的切片单位，不是套在每个工作上的模板。

### 关键文件

- **_本周.md**：活跃周工作台。装原始想法、任务清单、每日进展流水、孤儿文件登记。
- **_归档/**：历史周文件存为 `YYYY-Wnn.md`（如 `2026-W07.md`）。只读，供回溯查询。

## `_本周.md` 内部区块

每个 `_本周.md` 含以下区块：

### 1. 原始口述
未经过滤的原始输入——想法、问题、半成品计划、当天的特殊安排。AI 不编辑此区块，它是源材料。

### 2. 任务清单
可执行项，带项目上下文。AI 从口述生成，或推进中追加。格式：
```markdown
- [ ] [项目名] 任务标题 — 上下文/子任务
```
一周内随时更新：添加新项、完成打勾。继承项标 `[继承]`。

### 3. 孤儿文件
落在专注区、尚未找到归属的文件或想法。每条至少含名称 / 暂存理由 / 可能归属。

### 4. 进展记录
每日工作流水账，含项目索引。按日期节组织（`### 周X YYYY-MM-DD`），每段格式：

```markdown
**项目名称**（`项目根目录路径`）
关联：`具体文件1.md` · `具体文件2.md`

- 今天做了什么（流水账，不记理论推导细节）
- 确立了什么决策 / 推进到哪里
```

> 「关联：」字段是 `week-sync` 接续定位与 `weekly-review` 深度回溯的索引入口。
> 理论推导和结论细节记录在项目文件本体，此处只记做了什么。

### 5. 本周产出（归档时补写）
本周实际产出的文件汇总。**不在白天维护**——由 `weekly-review` 周日梦载荷在归档前一次性补写（见 §归档）。

## 孤儿文件机制

孤儿文件 = 落在工作区、还没找到家的零散件。它们的存在是合法的——只要还在 `_本周.md` 里登记，就在每次开机的视野内，不会丢。机制保证它们**每周被重新判一次去留**，不变垃圾场。

**核心区分：**
- **操作风险**：把孤儿移进某项目 / 素材库，是可逆动作（移错再移回）→ N 级，事后回执。
- **决策权**：孤儿该归哪个项目、要不要单独立项、还是扔素材库——这个判断要用户拍板。AI 只生成推理与建议，**不自作主张定家**。

**流转链：**

1. **周归档（当晚 dream 无人值守）**：没找到家的孤儿全部滚到下周 `_本周.md`——夜里不替用户猜归属。
2. **周一首会话（`week-sync`，只读）**：列出当前所有孤儿 + AI 主动推理每个可能属哪。纯提示建议，不动手。仅周一首会话触发，其余会话不重复提醒。
3. **用户拍板**：定家 / 扔素材库 / 再晾一周。
4. **执行移动（主会话，用户在场）**：拍板后由主会话当场移动，N 级回执。`week-sync` 只读、做不了移动。

> 软提醒：孤儿挂超过若干周仍无家，`week-sync` 提示时点一句"这个停留较久了"，建议定家或扔素材库当独立条目——只提醒，不强制清理。

## 归档

归档是**纯工作区维护**，不是"周末收尾仪式"：一张纸写满了 / 到周界了，换张新的，旧的存档。**白天不归档**——执行下沉到 `weekly-review` 周日梦载荷（当晚 dream 无人值守跑），严格顺序防丢内容：

1. `_本周.md` 补写 `### 本周产出`（本周实际产出汇总）。
2. `_本周.md` 重命名为 `YYYY-Wnn.md`（如 `2026-W26.md`），移入 `_归档/`。
3. 创建新的 `_本周.md`。
4. 未完成任务作为 `[继承]` 项进新周任务清单；孤儿文件随之滚入新周（见 §孤儿文件机制）。

**移入归档的**：已完成工作、已关闭任务（完成超一周）、历史参考。
**留在工作区的**：在制品、阻塞项、还没找到家的孤儿、当前周文件。

> 周回顾本身不靠本区域定义——它在 `weekly-review` / `week-sync` 自己那里，要的是进展记录里的流水。

## 日历生成（按需）

任务有时间承诺时，AI 可生成日历文件便于导入：
1. **识别**：扫描任务清单中的时间敏感项。
2. **提议**：建议时间块（时长、日期、冲突检查）。
3. **确认**：用户审批或调整。
4. **生成**：创建 `.ics` 文件，用户拖入日历应用。

## 区域参数

| 参数 | 值 | 说明 |
|---|---|---|
| 切片单位 | 周（`_本周.md` 一周一档） | 仅换纸与回顾的切片，非工作节律模板 |
| 归档触发 | 周界 / 用户手动 | 执行归 `weekly-review` 周日梦载荷 |
| 任务同步 | 实时 | 用户随时更新，AI 下次对话跟进 |
| 记忆写入 | 白天零写入 | 全部代谢归当晚 dream |
| 孤儿提示 | 周一首会话 | `week-sync` 列孤儿 + 推理归属，用户拍板 |
| 继承超时 | 1 周 | 过期任务下周自动提示决定 |

## 关键约定

- **链接**：库内相对路径（如 `../01 素材/参考.md`）。
- **状态标签**：`草稿`、`进行中`、`阻塞`、`完成`、`已归档`。
- **时间估算**：`(Xh)` 或 `(Xh + Yd)`。
- **标签**：`[项目名]` / `[主题]` 便于筛选。
- **_本周.md 写入**：静默写入，无需告知或确认，直接执行（AI 作为秘书做记录，不打断对话流）。
