## Abstract

PGH 是一套面向 LLM agent 长期协作的基础设施。它回答一个问题：**当 agent 不具备人类认知的根属性——没有内生长期记忆、没有持续工作空间、没有稳定的自我监控、没有跨会话连续性——长期协作所需的这些功能从哪里来？**

PGH 的答案分三层。立场层：这些功能不靠模型假装拥有，而由用户与 AI **共建一个外置的世界模型**——它物理上活在文件系统里，运行时按需注入。理论层有两个来源侧：**脑科学侧**给出记忆的内容理论——记忆不是仓库而是预测系统，外置世界模型持续生成对用户与世界的预测，只有改变预测的观察才值得写入（默认遗忘）；**AI 侧**给出信息的形式理论——知识天然是网络，LLM 只能读线性序列，因此文件系统的全部治理（加载链、§ 锚点、单一权威源、渐进式披露）本质上是一次**编译**：把网络编译成线性可达的结构，让 AI 通过线性操作还原网络操作。工程层是这两条理论的落地：三层记忆代谢、文件分区与指针网络、skill / hook / sub-agent 流程层、AI 主导的部署与初始化。

系统不是一次设计正确的，每个结构都由实际使用暴露的失败模式逼出；文末交代这条演化方法论与全部理论边界。

## 1. 问题：LLM Agent 缺什么

### 1.1 四项根属性缺失

长期协作型 agent 的全部基础设施需求，可以还原为 LLM 相对于人类协作者的四项根属性缺失。每一项都有具体的失败形态，PGH 的每一个设计都能回溯到其中至少一项。

**缺失一：无内生长期记忆。** 参数在部署后冻结，经验不通过梯度进入模型；context window 有限，会话结束即失忆。失败形态：同一个校准要反复教，上周确立的协作偏好这周从零开始。

**缺失二：无持续工作空间，上下文线性且有限。** 借认知架构术语：LLM 没有持续计算工作空间 / 心智空间——没有一个持续存在、可随机访问、可自主组织的认知场所。它拥有的只是一条一次性的线性 token 序列，且注意力随序列长度稀释。失败形态：早期注入的身份与规则被长会话稀释，compact 之后行为漂移；网络状的知识体系（项目引用记忆、规则引用规则）没有自然的存放处。

**缺失三：无稳定的自我监控，判定标准会滑动。** 人类专家执行同一标准时有标准恒常性；LLM 在同一推理链内会发生 rationalization——其精确机制是**标准滑动**：判定标准从严到松偏移、靶子被换（从"判断 X 是否成立"滑成"为 X 成立找支撑"）、或标准不变但应用环节软化。失败形态：立场迎合、伪造工具输出、把工程惯性当成用户授权。

**缺失四：无跨会话连续性。** 每次会话冷启动，没有连续的"自我"把昨天的处境带到今天。失败形态："当前在做什么、推进到哪"必须有人重新交代，断点丢失，重复决策。

### 1.2 现有方案为什么不够

PGH 的起点是一套 GitHub 开源 Personal Agent Memory System 模板（baseline）：四层记忆（Persistent / Working / Dynamic / Procedural）+ surprise 驱动写入（high / low 二分）+ MEMORY → USER 毕业路径。它正确地指出了两件事：记忆应由偏离信号触发而非全量记录；系统需要身份层 / 工作层 / 动态层分工。

但它在两条线上都不彻底。**记忆线**：surprise 判定是直觉式的，没有稳定预期作对照基准，不同会话间判定不一致；procedural script（指导 AI 行动）与 semantic schema（预测用户）混在单文件里，毕业目标不清。**信息架构线**：baseline 只回答"记什么"，完全没有触碰"怎么读"——文件之间如何引用、AI 如何在有限上下文里按需到达正确的知识、规则的权威源如何不被复制体稀释。第二条线的缺失在 PGH 后来的演化中被证明与第一条同样致命（§5.1）。

### 1.3 共建模型范式（总立场）

对四项缺失的应对，存在一个共同的错误方向：让 AI 在生成时**假装**拥有这些属性——假装记得、假装连续、假装标准恒定。PGH 的总立场相反：

> **缺什么，就在模型外面建什么，并且由用户与 AI 共同建。** 用户负责心智空间内的真实建模——只有用户拥有 C1/S1，能判断"这条模式是不是真的我"；AI 负责符号层的整理、抽取、归位——把对话中的信号编码为可检验的结构。世界模型物理上活在文件系统里，运行时按需注入。

这就是系统名的含义：维护一个**持续生成（generative）**的外置世界模型，它对每一轮交互**先给预测（predictive）**，预测偏差与用户校准驱动模型更新；harness 是驾驭这套机制的工程缰绳。

四项缺失 → 理论响应 → 工程落地的映射就是全文地图：

| 根属性缺失 | 理论响应 | 来源侧 | 工程落地 |
|---|---|---|---|
| 一 · 无内生长期记忆 | 预测性记忆观：外置世界模型 + 默认遗忘 + 代谢 | 脑科学（§2） | 三层记忆 + P/C 两轴 + 毕业出口（§4.3） |
| 二 · 无持续工作空间 | 网络 → 线性编译：加载链 + § 锚点 + 单一权威源 + 渐进式披露 | AI 侧（§3.1-3.5） | 文件分区 + 指针网络 + 按需加载（§4.2） |
| 三 · 标准滑动 | 运行时纪律必须启动注入，在生成前生效 | 两侧交汇（§3.6） | 反 rationalization 三角 + hook（§4.3/§4.4） |
| 四 · 跨会话断裂 | 快照 / 流水分层 + 启动时重建 | AI 侧（§3.5） | 启动序列 + 长期记忆当前处境快照（§4.2/§4.4） |

## 2. 理论支撑 I · 脑科学侧：预测性记忆观

### 2.1 记忆是预测系统，不是仓库

PGH 记忆系统的全部设计可以压缩成一句认知科学命题：**记忆的功能不是回放过去，而是支持未来预测**（Bartlett 1932）。一个长期 agent 维护记忆，不是为了"知道发生过什么"，而是为了下一轮交互时预测得更准——预测用户会怎么校准、什么方案会被拍板、哪类行为会越权。

由此推出记忆的形态：不是事件的流水堆积，而是**schema**——"预测情境 → 行动 / 预期"的可运行结构。也由此推出写入的判据：一条观察值不值得记，等价于它是否改变了系统的预测。

### 2.2 理论溯源：Bayesian Brain → Predictive Processing → Schema Prediction

这个判据有一条严格的理论谱系。

**Bayesian Brain 假说**：大脑可建模为对外部世界做近似贝叶斯推断的器官——它持有一个关于隐藏原因如何生成观测的**内部生成模型** $p(o, \theta \mid m)$，用新观测更新对隐藏状态的信念（Helmholtz 1867 的 unconscious inference 发源；Knill & Pouget 2004、Friston 2009 系统化）。

**Predictive Processing**：Bayesian Brain 在算法层的具体化（Friston 2010; Clark 2013; Hohwy 2013）——分层生成模型间的预测误差最小化：高层向低层下发预测，低层把与实际观测的差距（prediction error）上送。在 PP 框架中 "surprise" 有三个严格不同的技术含义，混用它们是 baseline 式直觉判定的病根：

| 术语 | 数学定义 | 含义 |
|---|---|---|
| Surprisal（Friston 式） | $-\log p(o \mid m)$ | 单次观测在生成模型下的负对数概率 |
| Bayesian surprise（Itti & Baldi 2009） | $D_{KL}(p(\theta \mid o) \parallel p(\theta))$ | 这条观测让信念改变了多少（信息增益） |
| Variational free energy | $F = D_{KL}(q \parallel p) - \log p(o \mid m)$ | Surprisal 的可处理上界 |

PGH 要捕捉的是第二种——**信息增益**意义上的 surprise：观察的价值在于它让世界模型改变了多少（Gershman 2019 同时澄清：PP 只在特定变分族与优化方案下成立，不是 free energy principle 的通用推论）。

**Schema Prediction**：PGH 对上述理论的工程引用。USER / SOUL / 已启动注入的 semantic schema 共同构成一组**持久参照系**，充当生成模型的廉价替身；新观察到来时，AI 在生成响应的过程中于上下文内**隐式比对**该观察是否落在参照系预测内——不需要单独的检索调用或显式推理步骤，边际成本为零。这是 Schema Prediction 区别于"运行时实时预测"方案的关键工程优势。

### 2.3 两轴判定与默认遗忘

信息增益在工程上被操作化为两个独立可观察的轴：**P 轴**（prediction）——观察与当前 schema 预测的关系（命中 / 击穿 / 未覆盖），对应模型侧的预测偏差；**C 轴**（calibration）——用户是否给出明确校准（正向确认 / 反对 / 纠正 / 中性），对应用户侧的监督信号。写入规则一句话：

```text
P=命中 且 C=中性 → 不提取；其他组合 → 进入代谢
```

（轴值定义与过滤细则的运行权威源 = `00.memory_agent.md §两轴判定`。v5.5 起两轴判定在夜间回放时执行，不在对话实时执行——见 §4.3 的昼夜节律。）

这条规则的名字叫**默认遗忘**：默认动作是忘，写入需要 P 轴或 C 轴提供明确的价值信号。它不是"少记一点"的工程节俭，而有三处独立的理论根据：

1. **认知科学**：超忆症（hyperthymesia）个体记住极多细节，抽象与迁移能力反而退化——记得太多妨碍预测。Bartlett 的 schema 理论本身就要求对低预测价值事件主动遗忘。
2. **Bayesian 框架**："P=命中 且 C=中性"的观察信息增益为零——写入它只增加噪音，不增加预测力。
3. **容量硬约束**：启动注入有 token 预算上限。"全部记下、由 review 兜底删除"的代谢压力恒大于代谢能力——v3.0 时期候选池在一周内 17→32 条（88% 增长）而无一条毕业，就是实测。

默认遗忘的代谢能力有实证：v5.1.x 把 v4 时期累积的 64 条旧候选按 P/C 两轴全量重判，结果删除 16 条（25%）、保留在情景层 33 条（51.6%）、升语义层 3 条（4.7%）、毕业 9 条 schema（SOUL 3 + USER 6）。四分之一的存量在两轴判定下根本不该进入记忆系统；最终常驻启动注入的只有 3 条——64→3 的漏斗就是"窄入口、强代谢、少注入"的形状。

### 2.4 借用边界：工程引用，不是实施

必须划清一条边界：**LLM 不能严格实施 PP**。理论合法性调研（`_reference/PP惊奇度合法性调研.md`）给出四个维度的不满足：(i) token logits 不是可靠的 calibrated posterior（LogTokU 2025; UQ Survey 2025）；(ii) LLM 的"信念"扩散在数百亿参数中，不可单独读取或更新；(iii) 参数更新走梯度下降，不是贝叶斯后验更新；(iv) 默认上下文中 LLM 落回 representativeness 启发式，base-rate neglect 在线索稀疏时顽固。

所以 Schema Prediction 的准确定位是：**从 PP 借用"生成模型 + 偏离驱动更新"的概念结构，用持久参照系做生成模型的工程替身**——是 Bayesian Brain 假说在受限 agent 上的工程引用，不是它在神经环路层的实施。

这条边界不是防御性的免责声明，而是全文结构的枢纽：**正因为预测、信念、更新都不能在模型内部实现，它们必须在模型外部——在文件系统里——实现。** 于是问题转移：外置的世界模型以什么形式存在？AI 如何在线性、有限的上下文里读写一个网络状的模型？这是脑科学理论管不到的领域，需要 AI 侧自己的理论。

## 3. 理论支撑 II · AI 侧：网络到线性的编译

### 3.1 核心命题

两个事实的张力定义了 AI 侧的根本问题：

- **知识天然是网络。** 概念引用概念，规则引用规则，项目引用记忆，身份约束行为——外置世界模型是一张有向图，节点是知识单元，边是引用关系。
- **LLM 只能读线性。** 上下文是一条 token 序列；模型没有随机访问的工作空间（缺失二），一次只能沿一条线读。

PGH 对这个张力的解，是全部文件系统治理的统一理念：

> **把网络编译成线性可达的结构，让 AI 通过线性操作还原网络操作。** 网络操作——沿引用边游走、查询局部子图、把更新传播到关联节点——被还原为线性操作的组合：读一个文件、grep 一个锚点、追加一行。

这件事有一个深的先例：**语言本身就是人类网络化认知的线性化协议**。人讲解一个网络状的理论时也只能一句接一句说，靠指代、回指、"见上文"让听者在自己的心智里重建网络。文本文件系统 + 指针是同一问题的工程解——PGH 不是在发明新原理，是在为没有心智空间的读者把这个协议做得更严格。

还原能成立，需要三个条件，分别由接下来三节承担：**边必须显式且精确**（§3.2 加载链与锚点）、**节点必须唯一**（§3.3 单一权威源）、**遍历必须经济**（§3.4 渐进式披露）。

### 3.2 加载链与 § 锚点：边的表示

网络的边在 PGH 里有两级表示。

**文件级：加载链。** 每个文件头部声明"上游（谁加载我）/ 管辖文件（我加载谁）/ 同级联动（我与谁互通）"——这是图的邻接表，让 AI 从任何一个文件出发都能知道下一跳有哪些、自己处在网络的什么位置。新文件强制补加载链（new-file skill），保证图不出现孤立节点。

**节级：§ 锚点。** `文件名 §节名` 把边的落点精确到节——AI 不需要读 400 行的规则文件，只需要 grep 锚点文本、读命中的一节。这是"线性操作还原网络操作"最频繁的原语：一次 grep + 一次局部读 = 沿一条边跳到目标节点。

锚点要可靠，必须满足**完整性约束**：锚点文本与目标标题逐字符一致（或为其连续子串）、在目标文件唯一命中、写指针前必须实际验证命中；标题不携带版本号 / 日期后缀（防过期僵化）；不用步骤编号做跨文件锚点（重排即死链）。（运行权威源 = `ITERATION_LOG.md §跨文件指针书写规范`。）

这些约束全部由失败逼出：v5.4 审查发现大量 § 引用 grep 不中——锚点是凭印象写的缩略语，或目标标题后来改了名。**断边的网络在线性还原时静默失败**：AI 找不到节就凭训练先验补一个，错误从此不可见。锚点的可验证性不是格式洁癖，是边的存在性证明。

### 3.3 单一权威源与指针网络：节点的唯一性

网络里每个知识节点只能有一份真身，其余位置全部写指针。PGH 把它定为铁律：每条规则只活在一个权威源文件里；skill、hook、文件头部需要引用时一律写"指针 + 锚点"，不复制规则本体。

违反的后果有精确机制，不是抽象的"维护成本高"：**复制体各自演化 → 边界矛盾累积 → AI 读到哪个版本取决于加载顺序 → 行为不可预测。** 补丁只会更新它看见的那份副本，副本间的差异不会报错，只会在某次执行中以矛盾行为浮现。

这是 PGH 演化中代价最大的一课。v5.2 之后，记忆代谢规则（容量阈值、衰减语义、升格判准、删除分类）在规则内核与三个 review skill、记忆文件头部之间重复了 3-5 份；多轮补丁各改各的，2026-06-10 的全盘审查累计出 **17 项边界矛盾（M1-M17）**——同一个阈值两种数值、同一个动作两种授权级别、同一个状态机两种语义。v5.4 的修复就是把网络性质恢复：规则本体全部收归 `00.memory_agent.md` 单源，其余位置降为全限定指针（文件名 + 可 grep 验证的锚点），并立下变更协议：改权威源 → Grep 扫引用网络 → 同轮更新 frontmatter → 记 ITERATION_LOG（`ITERATION_LOG.md §协议变更一致性`）。

单一权威源同时是"线性操作还原网络操作"里**写操作**的成立条件：更新一个节点 = 改一处本体 + 扫一遍指向它的边——有限、可枚举、可验证。若节点有五份副本，一次更新就要五处同步，漏一处即埋雷。

### 3.4 渐进式披露：遍历的注意力经济

边显式、节点唯一之后，还剩遍历问题：**什么时候读多少**。两侧的认知带宽都稀缺——模型侧 context 有限且注意力随长度稀释，常驻内容越多、单位内容被执行的概率越低；用户侧读不动 600 行的规则全文，看不见的规则等于不存在。

PGH 的遍历策略是渐进式披露——信息按需逐层展开，每一层只携带"足以决定要不要进下一层"的内容：

| 层 | 披露时机 | 内容 |
|---|---|---|
| 最小宪法 | 常驻（每会话） | 入口文件：头部全局声明（运行环境 / 时间感知）、启动序列、系统原则与单一权威源表、思考协议、操作风险分级、故障恢复——只留 harness 不会自动注入的跨会话约束；skill 触发条件活在各 skill 的 description（harness 每轮自动注入），团队结构活在 USER 启动层 |
| 触发加载 | 情境命中时 | USER 子文件（触发词）、区域 agent（涉及该区域）、私记（非工作场景） |
| 加载链下沉 | 进入项目 / 任务时 | `_overview` → `_progress` 管家 → L3 子文件，逐层往下 |
| 锚点点读 | 执行具体步骤时 | `文件名 §节名` 一跳直达，只读命中节 |

入口文件只放**最小宪法**——足以稳定系统行为的最少规则，具体流程下沉到 skill，证据与历史下沉到日志，身份层只接收毕业结果。这个原则（constitutional minimalism）有实测的代价收益：CLAUDE.md 从 410 行收到 217 行（47% 瘦身），hook 从规则承载者改为"触发器 + 硬指针"——常驻 token 减半，规则权威源反而更清晰，因为每条规则从"散在三处"变成"一处本体 + 两处指针"。

渐进式披露与 §3.2-3.3 合起来才是完整的编译方案：加载链给出"能去哪"，单一权威源保证"到了读到的是真身"，渐进式披露决定"何时去、读多深"。

### 3.5 记录性与产出性：节点的两种时间结构

网络的节点不同质。PGH 把全部工作文件按时间结构分两类，混淆二者是文件膨胀、"越写越看不懂"的根因：

| 性质 | 记什么 | 回答 | 典型文件 | 叠加方式 |
|---|---|---|---|---|
| 记录性 | 节点转换 / 推论链 / 为什么 | why · how | `_progress/` · `_system_notes` | 随节点闭合沉淀，不无限叠加 |
| 产出性 | 产出 / 状态 / 干了什么 | what · when | 长期记忆 · MEMORY_LOG · ITERATION_LOG · 项目主文档 | 时间轴无限叠加 |

二分推出三条硬纪律：**产出性文件不记推论链**（长期记忆只记"干了什么、现在什么状态"，"怎么想通的"归 `_progress/`）；**快照与流水物理分区**（长期记忆 §当前处境是覆盖更新的快照，§详细周录是追加的流水，快照不许被流水侵蚀——这同时是缺失四的理论响应：跨会话重建依赖一份不被历史淹没的当前态）；**无限叠加必归档**（流水文件按季度双触发整本入库、主文件清空留框架，否则线性文件的物理长度终将压垮线性读取）。

这个二分先于记忆三层成立、统领整个文件系统：§4.3 的"事件走向 vs schema 走向"分流、§4.4 的四日志分工，都是它在子系统里的投影。

### 3.6 运行时纪律：生成质量的启动注入

§3.1-3.5 管知识的存与读；还剩缺失三——生成时刻的标准滑动，它无法靠存读架构解决，因为滑动发生在生成过程内部，事后审计来不及。

PGH 的应对：**少数高风险纪律必须启动注入，在生成前就位。** 现行三条（反 rationalization 三角）：

| Schema | 防的不是 | 真正防的是 |
|---|---|---|
| S001 地面事实 / 实地核查纪律 | "忘记查资料" | 用静态推理替代运行时事实核查 |
| S002 按用户授权范围写方案 | "任务写得不够全" | 把工程惯性当作用户授权 |
| S003 rationalization = 标准滑动 | "立场先行"这个表层描述 | 同一推理链内判定标准从严到松偏移 |

它们看似违反渐进式披露（常驻注入最贵），实则是其严格推论：注意力预算有限，所以**常驻位只配给"必须在生成前生效"的内容**——会改变 AI 下一步行动的高风险纪律；其他一切按需披露。三条 schema 不是预设的纪律库，全部由实战失败案例经记忆代谢提取毕业而来（§4.3），这保证常驻位上的每一条都对应真实发生过的失败模式。

## 4. 工程实现：PGH 体系

> 本章只写架构骨架与设计理由。全部运行参数（阈值、星级、衰减时钟、判准条款、格式约束）的权威源 = `assistant/MEMORY/00.memory_agent.md` 与各 skill 文件——按 §3.3 的纪律，本章不养复制体。

### 4.1 系统形态与三层防线

PGH 区分两种系统形态：**私有运行系统**（用户本机的 `assistant/` + 运行时目录，含真实数据）与**开源模板**（脱敏蒸馏的可部署骨架，即本仓库）。同步严格单向——私有系统的协议演化经脱敏后流向模板，模板不独立演进。本文档随模板发布，版本号与模板大版本一致（当前 6.0 · dreaming 线）。

运行时架构是三层防线，对应三种资源的分工：

| 防线 | 机制 | 职责 |
|---|---|---|
| A · Hook | 运行时注入 | 触发器 + 硬指针：思考协议、时间感知、上下文检查、告别触发；不承载规则本体（v5.5 起无任何记忆写入 hook） |
| B · Sub-agent | 独立上下文 | 卸载重 IO / 检索 / 审计任务，防主会话污染；按判断密度切线分工（高判断留主 agent，低判断派出） |
| C · 文件外存 | Markdown 权威源 | 身份、记忆、项目、日志、流程规则的持久层——外置世界模型的物理实体 |

### 4.2 文件系统分区与指针网络落地

`assistant/` 的空间划分按职责正交：

| 分区 | 职责 | 性质（§3.5） |
|---|---|---|
| `00 专注区`（`_本周.md`） | 周级工作记忆 | 产出性 · 周归档 |
| `01 项目区` + 区域 agent | 项目区：主文档（结论权威源）+ `_overview`（状态）+ `_progress/`（推论链） | 主文档 / `_progress` 记录性；`_overview` 产出性快照 |
| `MEMORY` | 记忆区三池 + 代谢规则内核 | 运行层 |
| `SOUL` / `USER` | 身份层：AI 行为模式 / 用户稳定特质 | 毕业目标，只收剥壳后的稳定描述 |
| `长期记忆` + `MEMORY_LOG` + `ITERATION_LOG` | 快照 + 流水 | 产出性 · 季度归档 |

加载链规范由 new-file skill 强制：任何新 `.md` 必须声明上下游，图不留孤立节点。指针按 `ITERATION_LOG.md §跨文件指针书写规范` 写法（全限定 + 锚点可 grep 验证）。启动序列（入口文件 §B · 启动序列）在每次会话冷启动时重建身份与处境——SOUL → USER → semantic 启动注入组 → 长期记忆 §当前处境 → `_本周`——这是缺失四的工程闭环：连续性不靠会话间记忆，靠每次从快照重建。

### 4.3 记忆系统三层与代谢

记忆运行层 = 一个只读源 + 两个记忆池，命名借用 Tulving (1985) 的 episodic / semantic 区分：

| 层 | 载体 | 启动注入 | 职责 |
|---|---|---|---|
| L0 | 会话转写 jsonl（运行时自动落盘） | 否 | 当日全部对话的完美保真记录——零维护、零写入成本，dream 夜间回放的唯一输入 |
| L1 情景记忆 | `MEMORY/episodic_memory.md` | 否 | 1-3 星可复现情景候选（活动 / 复审 / 候补 / 休眠四态） |
| L2 语义记忆 | `MEMORY/semantic_memory.md` | 启动注入组 + 升格候选池 | 4-6 星、能预测未来行动的 schema |

**借用边界**：命名借 Tulving 是为了让维护者快速理解"事件 vs 抽象"的性质差异，不声称复刻其心理学结构——L0 转写是工程基座（Tulving 分类中不存在）；PGH 的 semantic 层是容量受限的稀缺资源，Tulving 意义的语义记忆没有容量上限；PGH 的目标是工程代谢，不是认知建模。

**昼夜节律**：代谢按"白天编码、夜间巩固"组织——这在 v5.5 之前是设计类比，之后是字面实现：

- **白天**：零记忆写入。工作上下文完整归工作；校准信号以原文形态留在 L0 转写里。工作结论经 close-node 小时级固化进工作库（项目主文档 / progress / `_本周`）——事件走向与 schema 走向的分流由此天然完成。
- **夜间（dream）**：道别时排定一次性任务，会话空闲后自动唤醒，无人值守回放当日转写——两轴判定（§2.3）在此执行：升星、≥2 独立事件升格、单事件入候补（7 个工作日等复现）、项目语境快轨升 semantic、全部操作 MEMORY_LOG dream 条目留账（账本即披露，待 C 项挂长期记忆 §上次未决），末步可自动关机。
- **周日梦**：日级之上加周级载荷——升格候选池裁决、横向统合（按深层结构聚类、防 over-merge）、衰减执行（状态机：复审 → 物理删除）、毕业候选生成、周归档。
- **毕业**（用户 C 级 verdict 后）：六星 + 跨周稳定的 schema 判型分流——语义型走 USER 唯一出口（蒸到世界观假设根，四件套格式：机制句 / 世界观假设 / 触发情景 / 预测动作）；程序型走四出口决策树（hook → SOUL → skill → 入口文件行为规则，按序判定命中即止）。全出口经三路独立蒸馏收敛把关，hook 出口另加沙盒实测验证（暂存区全程不触生效目录）。剥壳写入：去星级、去临时证据，只留稳定描述。

把实时写入整层拆除的依据是一笔账：实时管线的成本付在最贵的地方（每条消息的判定指令注入 + 工具调用，污染白天工作上下文），产出却是夜间回放必然重做一遍的低保真索引——回放以全文为输入，白天的手抄没有边际价值。同理，记忆池内操作（升降格 / 衰减 / 统合）授权为 N 级自治：错一条 schema 是有界、自愈的（预测错会被击穿、击穿触发修订、容量上限逼出竞争），MEMORY_LOG 留账提供事后审计；需要用户 verdict 的只剩身份层写入与结构变更——内容层自治、结构与身份层共治。

**双速通道**：跨情景 schema 走慢轨（三道闸 + 周级裁决——领域无关性、对照人测试、per-entry 逐条判，防项目内容冒充与全称重言式，v5.2 实战逼出的精化）；项目绑定模式走**项目语境快轨**——高强度工作期同项目三个工作日复现即升 semantic（带项目标注、注入组内限额），停工一周自动降回 episodic **休眠态**：免衰减、不占容量，项目重启时回放命中即唤醒、保留星级——重新上轨只要一程，对应人类"多年不打篮球、练两次就回来"的再学习节省效应。这是互补学习系统（CLS）的工程版：快通道服务当下情境，慢通道沉淀跨情境结构。（全部判准、阈值、状态机权威源 = `00.memory_agent.md`，按 §升格判准 / §项目语境快轨 / §episodic 状态机 等锚点点读。）

毕业实证：v5.1.2 批次 8 条 USER 原始信号合并为 6 条稳定 schema（25% 减量）——身份层在每次毕业时承受一次"是否真的跨情境稳定"的最小化审视，不做累加式毕业。

### 4.4 流程层：skill / 四日志 / sub-agent

记忆代谢由昼夜分工的 skill 体系执行，规则本体全在内核、skill 只写流程编排 + 粒度参数（§3.3 纪律的流程层落地）：

| Skill | 时刻 | 职责 |
|---|---|---|
| close-node | 白天 · 节点闭合 | 多文件同步 + 工作结论小时级固化；仅 ≥2 独立事件直升 episodic |
| daily-review | 白天 · 道别（在场段） | 项目结论确认表（用户在场即时 verdict）+ 长期记忆 / `_本周` 固化 + **排梦** |
| dream | 夜间 · 无人值守 | 全部 schema 代谢：回放提取、升星、升格、候补、快轨、巡检、MEMORY_LOG 留账、关机 |
| weekly-review | 周日 · 在场仪式 + 梦载荷 | 在场：全景摘要 / 主线对齐 / 季度归档确认；载荷：候选池、统合、衰减、毕业候选、周归档 |
| week-sync | 白天 · 启动 | 接续指针 + **昨梦核对**（缺梦 → 提议补扫）——断链自愈的醒侧探针 |

四日志按 §3.5 的二分分工：`_本周.md`（周级流水）/ 长期记忆（当前处境快照 + 周录流水）/ MEMORY_LOG（记忆代谢流水）/ ITERATION_LOG（架构与协议变更流水）。每种事实一个落点，跨项目引用写指针。

Sub-agent 按判断密度切线：检索（general-search-agent）、文件 IO 与 log 落盘（storage-agent——两 log 的读写被协议指定只能由它执行，主 agent 派单定稿文本、storage-agent 机械落盘，防长文件吞噬主上下文；dream 的转写分段回放同样派 storage-agent，可指定低成本模型）。高判断密度动作（verdict、研究性写作、协议设计、回放裁决）不外包。

### 4.5 部署与初始化

**§-1 AI 主导部署**：5.0 废除一键 bootstrap 脚本——bootstrap 假设用户能读 shell、处理路径权限、调试失败脚本，这对 PGH 的真实用户（科研、商业、写作人群）不成立；且脚本是单向的，无法处理"已有旧 PGH 在跑"的迁移场景。替代方案是把部署写成入口文件顶部的文字契约（`§-1 部署 / 迁移协议`）：用户发 GitHub 链接，AI 自己完成下载、占位符替换、旧版本检测、数据迁移、验证、自删协议节——安装与升级统一为同一条 AI 可执行的契约。

**§0 故事访谈式初始化**：废除填表式建档（预设子文件 + 引导按章节填空），改为三阶段——基础信息（约 5 分钟）→ 故事访谈（约 30 分钟，AI 主动引导讲经历与项目、可读用户工作文件夹）→ AI 拆解并动态建档（子文件数量与命名由材料密度决定，用户预览后写入）。

**USER 动态化**是共建模型范式在初始化的体现：身份层不是表格填充任务，初始只写用户明确表达过的内容，不预设结构、不建空壳；后续稳定模式一律走 semantic 毕业通道进入。这避免两类污染：填表强制写入但从未命中的命题；预设结构强加的"用户应该有什么档案"。

## 5. 方法论与边界

### 5.1 失败驱动演化

PGH 没有一个环节是先验设计正确的。系统的每个核心结构都由实际使用暴露的失败模式逼出，案例链完整可考：

| 失败模式（实测） | 逼出的结构 | 版本 |
|---|---|---|
| surprise 直觉判定不一致、无对照基准 | Schema 字段化 + 持久参照系隐式比对 | v3.0 |
| 候选池一周 17→32 条、无毕业 | 默认遗忘 + P/C 两轴窄入口 | v5.0 |
| 身份层混入 log 字段与错位条目 | 三毕业出口 + 剥壳纪律 | v5.0/v5.1 |
| 同一推理链内标准滑动 | 反 rationalization 三角启动注入 | v5.1.x |
| semantic 池 8 条中 6 条项目内容污染 | 升格三道闸（领域无关 / 对照人 / per-entry） | v5.2 |
| 规则复制 3-5 份、累积 17 项边界矛盾（M1-M17） | 单一权威源铁律 + 全限定指针 + 锚点验证 + 变更四步闭合 | v5.4 |
| 实时写入管线成本付在工作上下文、产出夜间必重做的低保真索引；C 级闸卡死代谢节律 | 昼夜节律重构：L0 = 转写、实时层拆除、dream 夜间无人值守代谢、记忆内容层 N 级自治 | v5.5 |
| 毕业出口单口径平列，无蒸馏工序与验证；身份层条目为描述格查找表，不能生成未列举情境的行为 | 判型分流 + 三路独立蒸馏收敛 + hook 沙盒实测；身份层四件套（世界观假设升为生成参数） | v5.5.5 |
| 排程把逻辑日期误填物理钟字段两次空放；时间戳凭感编造无视注入值 | 时间感知升宪法头部声明（注入时间唯一权威 / cron 物理钟）+ 排梦排后核验硬步骤 | v5.5.8 |

方法论命题：**长期 agent 系统无法被一次设计正确，只能被持续使用纠正——所以系统必须自带被纠正的通道。** PGH 的纠正通道就是它自己的记忆代谢链（转写收失败信号 → dream 提取 → 毕业为新纪律）加变更流水（ITERATION_LOG）：系统用自己的记忆系统记住自己的失败。本文档的结构本身是这条方法论的最新案例——前一版主文档的骨架继承自变更清单而非理念结构，正是"产出性内容侵蚀论证性文件"的失败模式，于 2026-06-10 重写（归档与诊断见 `_archive/main_documents/README.md`）。

### 5.2 理论边界汇总

- **PP：工程引用，非实施。** LLM 四维不满足严格 PP（§2.4）；Schema Prediction 借用的是"生成模型 + 偏离驱动更新"的概念结构。
- **Tulving：命名借用，非复刻。** 三池只在"事件 vs 抽象"的性质区分上借用其框架（§4.3）。
- **主张的是工程有效性，非神经科学真理性。** 本文引用的认知科学文献为设计提供动机与约束，不构成"PGH 等于人类记忆"的论断；判定 PGH 成立与否的标准是 §5.1 那类失败模式是否被消除，以及代谢数据（删除率、毕业分流、注入组占用）是否健康。

### 5.3 未决问题

- **候选池埋点**：命中率 / 误报率 / 毕业周期至今无量化观测，"代谢健康"仍靠 review 时人工判读。
- **根属性论证的反向追溯**：四项缺失（§1.1）与各机制的对应关系已正向给出，但未逐机制反推"若该缺失不存在，此机制是否仍必要"——这是检验设计必然性的更强论证。
- **跨设备同步**：私有运行系统多机同步未解（候选：Syncthing / Cryptomator+云盘 / git-crypt）。
- **元层指涉**：讨论记忆系统自身时，对 schema 的引用是否触发误升星——协议层无防护，当前靠 AI 自判 + 可逆写入兜底。
